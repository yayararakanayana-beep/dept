# Task 2-8d: 地形改変作用 標的化・強度・期間制御 設計仕様 RC1

## 0. 位置づけ

Task 2-8cでは、地形改変作用の方向性を最小検証した。

結果として、地形改変作用を広く・粗く入れるだけでは、副作用が勝ちやすいことが分かった。

したがってTask 2-8dでは、地形改変作用の有効性そのものではなく、以下を検証対象にする。

```text
どの条件なら、地形改変作用が副作用込みで長期正味利益を改善するか。
```

本タスクでは、最初から完全な非恣意性を狙わない。

まずは意図的に条件を絞り、改善が出る可能性のある領域を見つける。

---

```text
対象状態:

  対応する力学だけに作用

固定検証条件:
  強度: 低 / 中 / 高
  期間: 短 / 中 / 長

評価:
  長期正味利益がプラスになる条件を探す
```

Task 2-8dでは、以下の恣意性を一時的に許容する。

```text
高リスク判定閾値
ピーク高判定閾値
トレンド悪化判定閾値
v8局所観測の一致判定

```

理由:

```text
まず改善条件の存在を探すため。
```

ここで改善条件が見つかった場合、後続タスクで閾値・強度・期間を較正する。

---

## 4. 標的化の基本条件

地形改変作用は、次の3条件のうち2つ以上を満たす場合だけ候補化する。

```text
risk_level >= 0.65
risk_slope >= 0.45
peak_risk_estimate >= 0.70
```

ただし、次の場合は例外的に候補化を許す。

```text
risk_acceleration >= 0.60
かつ
v8局所観測が自己増幅または入力過敏を示す
```

目的は、地形改変作用を広く撃たず、高リスク化しやすい領域に絞ることである。

---

## 5. 力学一致条件

地形改変作用は、v8局所観測の原因力学と一致する場合だけ使う。

### 5.1 入力感度低下作用

使用条件:

```text
下位リスク情報:
  input_sensitivity_score >= 0.58
  risk_slope >= 0.40

v8局所観測:
  input_overreaction
  または boundary_instability
```

作用対象:

```text
入力 → リスク上昇
```

### 5.2 自己増幅遮断作用

使用条件:

```text
下位リスク情報:
  amplification_score >= 0.58
  risk_acceleration >= 0.42

v8局所観測:
  recursive_amplification
  または oscillation
  または recurrence
```

作用対象:

```text
リスク上昇 → さらなるリスク上昇
```

### 5.3 回復谷形成作用

使用条件:

```text
下位リスク情報:
  recovery_margin <= 0.42
  irreversibility_risk >= 0.52

v8局所観測:
  recovery_failure
  または regime_switch
  または split_merge_return_failure
```

作用対象:

```text
高リスク状態 → 回復方向への勾配
```

## 6. 強度条件

Task 2-8dでは、強度を3段階に固定する。

```text
低: 0.18
中: 0.30
高: 0.42
```

仮説:

```text
低:
  副作用は小さいが、地形変化が弱い

中:
  最も有望

高:
  リスク改善は大きいが、副作用が勝つ可能性がある
```

目的は、最適値ではなく、改善領域の粗い位置を探すことである。

---

## 7. 期間条件

Task 2-8dでは、作用期間を3段階に固定する。

```text
短: 12 step
中: 24 step
長: 36 step
```

仮説:

```text
短:
  副作用は小さいが、持続改善が残りにくい

中:
  最も有望

長:
  地形変化は強いが、副作用が蓄積しやすい
```

評価では、作用中の改善だけでなく、作用停止後の改善持続を見る。

---

## 8. 成功条件

Task 2-8dで見たい成功条件は、以下である。

```text
long_term_net_benefit > 0
risk_peak_delta_vs_no_op > 0
risk_auc_delta_vs_no_op > 0
side_effect_delta_vs_no_op は許容範囲内
post_action_persistence > 0
```

特に重要なのは、以下である。

```text
副作用はあるが、長期正味利益がプラス
```

つまり、地形改変作用の副作用をゼロにすることは目的ではない。

副作用込みで、長期利得が勝つ条件を探す。

## 9. 検証比較

Task 2-8dでは、以下を比較する。

```text
A. no_op
B. 状態作用のみ
C. 地形改変作用: 標的化なし
D. 地形改変作用: 標的化あり
E. 状態作用 + 地形改変作用: 標的化あり
```

目的:

```text
標的化がない地形改変作用は副作用が勝つか
標的化を入れると改善するか
状態作用との併用が有効か
```

Task 2-8dには入れないが、探索軸の役割は次のように想定する。


## 12. 実装方針

Task 2-8d実装では、以下を行う。

```text
1. Task 2-8b候補を入力にする

3. v8局所観測と作用種別を対応させる

6. 長期正味利益がプラスになる条件を抽出する
```
