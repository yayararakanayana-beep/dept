# Task 3.2-4.1b 軽量感度・原因切り分け監査

## 目的

Task 4.1の修正範囲を、既存正式成果物の再集計と500分岐以内の追加監査で確定する。

## 実行段階

1. Phase A: 既存fit/validation 72状態の再集計
2. Phase B: 構造量から固定した代表18状態で4種類の継続仮定を比較
3. Phase C: 判定差または既存成功・失敗混在がある最大12状態だけ作用感度監査
4. Phase D: 成功・失敗境界がある最大6状態だけ遅延・強度を細分化
5. Phase E: 複製不一致がある最大4条件だけ追加複製

## 継続仮定

- C0: 現在入力を8ステップ固定後、中立化
- C1: 即時中立化
- C2: 元軌道の未来入力を再生
- C3: 同seedの安定継続入力を再生

## 予算

- Phase B: 144分岐
- Phase C: 最大144分岐
- Phase D: 最大72分岐
- Phase E: 最大40分岐
- 絶対上限: 500分岐

## 境界

- holdout状態ファイルを読まない
- 予測器を再学習・再選択しない
- 正式構造正解を置き換えない
- 元軌道へ書き戻さない
- Parameter Boxへ書き込まない
- Action Moduleへ接続しない
- scenario IDを代表状態選択に使わない
