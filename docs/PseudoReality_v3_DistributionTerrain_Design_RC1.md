# PseudoReality v3 Distribution Terrain Design RC1

## 1. 目的

PseudoReality v3 は、個々のプレイヤーを精密に動かすための世界ではない。
v3 は、プレイヤー群を状態分布として扱い、その分布が利得地形の上を流れ、流れ自体が次の地形を変形させる、軽量な疑似オープン現実システムである。

v3 の目的は、現実システムに現れうるマクロなゲーム構造らしい歪みを、次の要素から軽く再現することである。

- 状態分布
- 短期利得地形と中期利得地形
- 外部要因
- 粘性、摩擦、回復性
- 非線形しきい値
- 累積した流れによる地形変形

v3 は、縮小均衡、探索低下、情報品質劣化、不可逆化、回復困難、外部ショックによる地形変形、未使用領域の細り、過集中ダメージなどを、直接の現象名ラベルやフラグとして実装しない。これらは、利得勾配、流れ、摩擦、粘性、しきい値、地形変形の相互作用から発生するマクロ動態として扱う。

## 2. v2 から v3 への転換

PseudoReality v2 は、個体に近い疑似世界として設計されていた。v2 では、個別エンティティ、公開状態、隠れ状態、関係トレース、ゲームトレースが中心であった。

v3 は、v2 の個体をさらに複雑にする方向には進まない。v3 の設計転換は次の通りである。

- 旧方向: 個別エンティティ更新モデル
- 新方向: 状態分布と利得地形の相互更新モデル

v3 で重要なのは、個々のプレイヤーの正確な行動予測ではない。重要なのは、集合的な利得追求行動から現れるマクロな流れ、歪み、盆地、固定化、集中、回復困難である。

## 3. 中心イメージ

v3 は、自己変形する分布地形モデルである。

- 状態空間 = 地形
- 短期利得 / 中期利得 = 勾配
- プレイヤー人口 = 粘性を持つ液体
- 分布移動 = 利得勾配に沿った流れ
- 慣性 / 摩擦 = 粘性
- 外部要因 = 地形を傾ける、または変形させる外力
- 累積した流れ = 地形変形

中心ループは次の通りである。

```text
地形
  ↓
分布の流れ
  ↓
地形変形
  ↓
次の流れ
```

このループにより、v3 は「状態分布が利得地形を流れ、その流れが次の利得地形と移動条件を変える」疑似現実として動く。

## 4. v3-0 の非目標

v3-0 は設計文書の段階であり、次を実装または詳細設計しない。

1. `G_t` 接続
2. `O_t` 接続
3. DEPT 閉ループアクション
4. 個別プレイヤーの精密予測
5. 個体ごとの係数差
6. 適応的なプレイヤー型比率
7. 複雑なアクション遅延キュー
8. 汎用アクション難易度モデル
9. 次のような現象名フラグ
   - 縮小均衡フラグ
   - 探索低下フラグ
   - Goodhart フラグ
   - 崩壊フラグ

`G_t` 接続は後続タスクで扱う。この文書は、PseudoReality v3 そのものの疑似オープン現実モデルに集中する。

## 5. コア変数

状態分布 (`D_t`):
: 状態空間上に存在するプレイヤー群の粗視化分布。個別プレイヤーではなく、各状態セルにどれだけの人口または重みが存在するかを表す。

地形 (`H_t`):
: 状態空間全体の地形。利得、摩擦、粘性、回復性、損傷などを含む、分布が流れる場の総称である。

短期利得地形 (`S_t`):
: 直近の利得を表す地形。資源獲得、競争回避、短期防衛、短期抽出などの魅力を表す。

中期利得地形 (`M_t`):
: 数ステップ先の維持可能性、探索余地、回復可能性、情報品質などを反映する利得地形。短期利得と一致するとは限らない。

外部要因 (`C_t`):
: 外部資源供給、需要、競争圧、情報ノイズ、ショック、制約圧など、地形や流れを外側から傾ける入力。

分布流 (`F_t`):
: 状態分布がどの方向へ、どの程度移動するかを表す流れ。集中、停滞、拡散、逆流しにくさを含む。

粘性 / 慣性 (`I_t`):
: 分布の動きにくさと反転しにくさ。高い場合、分布は環境変化に即応せず、既存の流れを維持しやすい。

非線形しきい値 (`B_t`):
: 資源、圧力、情報品質、損傷、硬直などが一定水準を越えたとき、流れ、摩擦、回復速度、利得地形を急変させる境界。

地形変形 (`U_t`):
: 外部要因と累積した分布流により、次時刻の地形が変化する量。経路強化、過集中ダメージ、自然回復、未使用領域の細りを含む。

## 6. 主計算軸

RC1 では、状態分布 `D_t` が実際に流れる主計算軸を次の 5 つに限定する。

1. 資源余裕 / Resource slack
2. 情報品質 / Information quality
3. 圧力 / Pressure
4. 探索余地 / Exploration room
5. 可逆性 / Reversibility

これらの 5 軸を優先する理由は、現実システムにおけるマクロなゲーム構造と直接関係するためである。資源不足、情報劣化、競争圧、探索余地の喪失、回復困難を表現できる。

v3 RC1 は主軸を増やしすぎない。格子の大きさは軸数に対して指数的に増加するためである。推奨開始点は次の通りである。

```text
5 軸 × 各 5 ビン = 3125 セル
```

これは実装開始時の推奨値であり、最終要件ではない。

## 7. 補助属性

補助属性は、分布がどのように流れ、回復し、固着し、広がり、歪むかに影響する地形側の性質である。補助属性は直接の現象ラベルではない。

候補となる補助属性は次の通りである。

- 疲労 / fatigue
- 損傷 / damage
- 硬直 / rigidity
- 結合度 / coupling
- 摩擦 / friction
- 粘性 / viscosity
- 回復速度 / recovery speed
- 局所伝播性 / local propagation

悪い使い方は、縮小均衡フラグ、探索低下フラグ、Goodhart フラグ、崩壊フラグのように、結果名を直接与えることである。

良い使い方は次の通りである。

- 高い疲労が、探索方向への流れを弱める。
- 高い損傷が、可逆性の回復を遅くする。
- 高い硬直が、既存の盆地からの脱出を難しくする。
- 高い結合度が、圧力や情報劣化を局所的に伝播させる。
- 高い摩擦が、遷移コストを増やす。
- 高い粘性が、分布移動を遅くする。

補助属性は原因側の地形性質である。補助属性は DEPT に答えを教えてはならない。

## 8. 外部要因

v3 RC1 で扱う外部要因は次の通りである。

- 外部資源供給 / external resource supply
- 外部需要 / external demand
- 外部競争圧 / external competition pressure
- 外部情報ノイズ / external information noise
- 外部ショック / external shock
- 外部制約圧 / external constraint pressure

外部要因は現象名ラベルを直接作らない。外部要因は、短期利得地形、中期利得地形、摩擦、粘性、回復速度、局所伝播性、非線形しきい値に作用する。

例:

- 外部資源供給が減ると、資源余裕の地形が悪化し、短期利得が抽出や囲い込みへ傾きやすくなる。
- 外部情報ノイズが増えると、中期利得の見積もりが歪み、防衛的または待機的な流れが選ばれやすくなる。
- 外部競争圧が増えると、短期利得の重みが増え、探索や協調が選ばれにくくなる。
- 外部制約圧が増えると、特定方向の摩擦が上がり、分布流が変化する。

## 9. 分布流の概念計算

分布は複合利得勾配に沿って流れる。概念的には次のように考える。

```text
複合利得
  = 短期利得重み × 短期利得地形
  + 中期利得重み × 中期利得地形
  - 摩擦
  - 不確実性
```

分布は、より高い複合利得へ向かって移動する。ただし、粘性と慣性があるため瞬間的には移動しない。

- 粘性が高い場合、分布はゆっくり動き、既存の流れは反転しにくい。
- 粘性が低い場合、分布は外部環境の変化に素早く反応する。

v3-0 では厳密な数式は定めない。ただし v3-1 実装では、各セルから近傍セルへの候補遷移を評価し、複合利得差、摩擦、粘性、しきい値、拡散成分に基づいて `F_t` を更新する方針とする。

## 10. 地形変形

地形変形は v3 の最重要要素である。分布流は次の地形を変える。

概念的な地形更新は次の通りである。

```text
次の地形
  = 現在の地形
  + 外部要因による変形
  + 流れによる経路強化
  - 過集中による損傷
  + 自然回復
```

例:

- 流れがある経路に集中すると、その経路は使いやすくなる。しかし過剰に使われると、資源が減り、損傷が増え、可逆性が低下する。
- 未使用領域は、資源や探索余地を回復する場合がある。しかし長期間使われない場合、経路が細り、再進入の摩擦が上がる場合もある。
- 短期利得の盆地が分布を引き寄せると、流れが集中する。その結果、中期利得が悪化し、盆地が深くなり、脱出が難しくなる。

この設計により、縮小均衡、過集中、探索低下、回復困難は、直接ラベルなしで発生しうる。たとえば、短期利得へ流れが集中し、探索経路が細り、損傷と硬直が増え、可逆性回復が遅くなると、外から見れば縮小均衡に似た状態が現れる。

## 11. 非線形しきい値

非線形しきい値 `B_t` は、状態や補助属性が一定範囲を越えたとき、地形、摩擦、回復速度、流れを急に変化させる。

例:

- 資源余裕がしきい値を下回ると、囲い込みへの短期利得が急上昇する。
- 情報品質がしきい値を下回ると、防衛的または待機的な流れが増える。
- 圧力がしきい値を上回ると、可逆性が失われやすくなる。
- 探索余地がしきい値を下回ると、新しい経路に入りにくくなる。
- 損傷がしきい値を上回ると、回復速度が急落する。
- 硬直がしきい値を上回ると、逆方向移動への摩擦が急上昇する。

これらのしきい値は、現象名フラグを直接設定しない。あくまで地形、摩擦、回復速度、流れを変化させる原因側の機構である。

## 12. アクション扱い

v3 RC1 では、アクション扱いを単純に保つ。複雑なアクション遅延、アクションキュー、汎用アクション難易度は実装しない。

理由は、v3 RC1 の目的が自己変形する分布地形型の疑似現実を作り、検証することだからである。アクション機構そのものを難しくすることは目的ではない。

将来、システム固有の入力として、アクションが地形または外部要因へ直接作用する設計はありうる。しかし、アクション統合は v3-0 の範囲外である。

## 13. トレース設計

将来実装では、次のトレースカテゴリを提案する。

### v3 世界内部用

`v3_internal_distribution_trace`:
: 内部状態分布 `D_t`。

`v3_internal_terrain_trace`:
: 地形 `H_t`、短期利得 `S_t`、中期利得 `M_t`。

`v3_internal_flow_trace`:
: 分布流 `F_t`、集中、停滞、拡散。

`v3_internal_external_trace`:
: 外部要因 `C_t`。

`v3_internal_auxiliary_trace`:
: 疲労、損傷、硬直、摩擦、粘性、回復速度、局所伝播性。

### 将来の観測 / DEPT 接続用

`v3_public_observation_trace`:
: v3 から外部へ出す圧縮公開観測。

`v3_gt_confidence_distribution_trace`:
: 将来、公開観測から生成される `G_t` 候補。

`v3_gt_history_trace`:
: 将来の `G_t` 履歴。

`dept_inferred_structure_trace`:
: 将来の DEPT 側推定マクロ構造トレース。

### 検証専用

`v3_hidden_structure_diagnostic_trace`:
: 検証専用の隠れ診断トレース。DEPT 入力として使ってはならない。

`G_t` 接続は本タスクの範囲外である。

## 14. 可視性と漏洩防止ルール

世界内部トレースは、シミュレーションのデバッグと検証のために存在する。将来の DEPT 統合において、世界内部トレースを直接 `G_t` または `O_t` 入力として使ってはならない。

将来の `G_t` は公開観測のみから生成されるべきである。将来の `O_t` は `G_t` / `K_t` を読むべきであり、世界内部を直接読んではならない。

これは、トレース漏洩によって DEPT に答えを教えてしまうことを避けるための設計ガードレールである。

## 15. 既存手法との関係と実装方針

関連する既存手法には次がある。

- 平均場ゲーム / mean field games
- 移流・拡散・反応方程式 / advection-diffusion-reaction equations
- システムダイナミクス / system dynamics
- エージェントベースモデル / agent-based modeling

ただし v3 RC1 では、重い外部シミュレーション基盤を導入せず、軽量な NumPy / pandas 実装を推奨する。

理由は、v3 RC1 を検査しやすく、既存の v2 コードベースに接続しやすく、トレースしやすく保つためである。

将来の比較または拡張候補は次の通りである。

- FiPy: より形式的な移流・拡散モデル用
- Mesa / NetLogo: 個別エージェント比較用
- PySD: stock-flow モデリング参照用
- 平均場ゲーム系ライブラリ: 理論比較用

## 16. 将来実装の成功条件

将来の v3 RC1 実装は、少なくとも次を満たすことを成功条件とする。

1. 外部要因を変えると分布流が変わる。
2. 短期利得と中期利得の不一致が、分布の歪みとして蓄積する。
3. 分布流が後続ステップの地形を変える。
4. 補助属性が、現実的な摩擦、粘性、損傷、硬直、回復困難を作る。
5. 非線形しきい値が、急変、固定化、回復困難を生成する。
6. 短期集中、縮小均衡、探索低下、情報劣化、回復困難などのマクロ動態が、直接の現象フラグなしで発生する。
7. 後続の DEPT 統合は、これらの歪みを公開観測と派生した `G_t` / `K_t` を通してのみ読む。世界内部トレースを直接読まない。
8. 将来の DEPT 由来のシステム固有アクションは、無操作や単純な短期利得追従と比較されてもよい。ただしこれは v3-0 の範囲外である。

## 18. RC1 設計境界のまとめ

PseudoReality v3 RC1 は、自己変形する分布地形モデルとして凍結する。個体差、個別係数、個別プレイヤー予測、複雑なアクション機構、`G_t` / `O_t` / DEPT 接続は v3-0 の範囲外である。

v3 の中心は、状態分布、短期・中期利得地形、外部要因、補助属性、非線形しきい値、地形変形の相互作用である。この相互作用により、マクロなゲーム構造らしい歪みが、直接ラベルではなく原因側の地形動態から現れることを目標とする。
