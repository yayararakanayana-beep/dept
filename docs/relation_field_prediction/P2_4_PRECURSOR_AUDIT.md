# P2-4 連続前兆座標監査

## 目的

P2-4は、P2で構築した連続座標が、複雑な予測器を学習する前の段階で、RF-10の未来観測結果に対する先行情報を持つかを監査する。

P2-4自身は予測器ではない。重み学習、閾値調整、候補選択、作用選択を行わない。

## 時間境界

処理は必ず二段階に分ける。

```text
1. 接頭部だけからP1・P2を再検証
2. cutoff_tのP2座標、差分、P1候補を凍結
3. 予測側ハッシュを確定
4. その後で初めて完全軌道の未来接尾部を読む
5. RF-10固定定義で未来観測結果を生成
6. 前兆指標を計算
```

未来接尾部、シナリオ、seed、作用、事象、世界内部状態は予測側へ入れない。

## 未来結果

RF-10の定義を変更せず再利用する。

| P2構造 | RF-10未来結果 |
|---|---|
| 過収束 | 集中進行 |
| 固定化 | 集中持続 |
| 発散 | 分散進行 |
| 回復余地低下 | 回復失敗 |

未来幅は1、2、4遷移後。回復失敗はRF-10が適用可能と判定した標本だけを評価する。

## 比較する得点

- P2構造余裕
- P2構造余裕の一階変化
- P2構造余裕の二階変化
- 構成要素だけの連続余裕
- P1の同時点ブール候補
- 常に候補なし
- 分割内で決定論的に時系列対応を崩したP2構造余裕

四構造は別々に評価し、単一危険度へ合算しない。

## 指標

閾値を調整せず、主に次を保存する。

- ROC曲線下面積
- 平均適合率
- 陽性率
- 陽性・陰性の得点平均と中央値
- ゼロ境界での混同行列指標
- 軌道群単位の決定論的再標本化95%区間

重複未来窓を独立標本とは扱わない。

## 支持条件とPhase3移行

RF-10の支持条件を継承する。

- 未知testケース30件以上
- 各構造・各未来幅で陽性5件以上
- 各構造・各未来幅で陰性5件以上

支持不足時は、性能が高い・低いという判定を行わず `blocked_support_insufficient` とする。

支持条件を満たした後、P2構造余裕について、少なくとも2つの未来幅で、再標本化区間の下限が次を同時に満たす場合に、その構造をPhase3比較対象とする。

```text
ROC曲線下面積 > 0.5
平均適合率 > 陽性率
```

全4構造で成立すればPhase3全面比較、一部だけなら該当構造に限定した部分比較へ進む。

## 適用可能性情報

候補幅、未解決残差比、履歴被覆度、識別子安定度、座標利用可能率は層別診断に使う。未来結果の定義変更や標本除外の後付けには使わない。

## 小規模監査

専用CIでは、3つの共通接頭部へ異なる未来を2本ずつ接続した6ケースを生成する。

この監査の目的は、

- 同じ接頭部なら予測側が同じ
- 未来だけ変えると結果側だけが変わる
- P1→P2→P2-4が一気通貫で動く
- 指標と支持条件を独立再計算できる
- 小標本で性能主張をしない

ことの確認である。6ケースは正式な前兆性評価件数を満たさないため、科学判定は支持不足のCに留まる。
