# P2-4 正式前兆監査 v2

## 目的

P2連続座標が、RF-10の固定未来結果に対して前兆情報を持つかを、
予測器学習前に漏洩なく監査する。

本監査は合成軌道に限定される。現実データ、未知軌道族、因果予測、
真の不可逆性、作用推奨は主張しない。

## v1からの修正

### 1. 回復余地低下の未来幅1

RF-10の回復失敗は未来幅2以上でのみ適用可能である。

- 過収束・固定化・発散: 未来幅1、2、4
- 回復余地低下: 未来幅2、4
- 回復余地低下・未来幅1: `not_applicable_by_contract`

未来結果定義や閾値は変更しない。

### 2. P2単一値座標の読取り

P2の登録済み単一値座標は、保存上 `shape == (1,)` である。
旧P2-4読取り処理は0次元配列だけを受け入れたため、存在する座標を
`None` と判定していた。

v1.1 builderは次だけを単一値として受け入れる。

- `shape == ()`
- `shape == (1,)`

それ以外の配列を暗黙に平坦化しない。

### 3. 監査順序

正式順序は次で固定する。

1. 接頭部だけからP1・P2を生成・検証
2. cutoff時点の予測スナップショットを凍結
3. 四構造のP2主座標が全ケースで利用可能か検査
4. 利用可能性検査に成功した後で初めて未来接尾部を読む
5. RF-10未来結果を生成
6. 指標・支持条件・Phase3判定を計算
7. 独立検証器で再計算

主座標が欠落する場合、未来接尾部は読まない。

## データ独立性 v2

36ケースすべてに、軸置換・鏡映前の異なる接頭部力学を持たせる。

v2計画は、履歴保存されたv1計画のケース割当・未来族・変換を基礎に、固定アルゴリズムで36個の接頭部プロファイルを生成する。生成直後に正規化SHA-256 `ce23c17576007eb3eab394feeb4fe716b8d9215c25cbfb8f5b074fee5c2052f7` と照合し、一致しなければP1/P2生成前に停止する。

必須条件:

- 36ケース以上
- 1ケース1軌道群
- 36個の固有接頭部プロファイル
- 直近4フレームの異なる力学を30種類以上
- 軸置換・鏡映だけを独立ケースに数えない
- 同一接頭部へ未来だけを付け替えた対は主性能監査から除外
- P1/P2得点を見る前にデータ計画を固定
- 得点を見たケース選択を禁止
- 全ケースをtest分割とする
- RF-10期待結果と実結果を完全照合する

接頭部分布は `point`、`narrow`、`broad`、`wide`、`boundary`
の明示的なフレーム列として固定する。

## 支持条件

- testケース30件以上
- 適用可能な各構造・未来幅で陽性5件以上
- 適用可能な各構造・未来幅で陰性5件以上
- 必須セルは11件
- 回復余地低下・未来幅1は契約上適用不能

## 判定

- `blocked_support_insufficient`
- `blocked_no_supported_precursor_signal`
- `eligible_for_partial_phase3_model_comparison`
- `eligible_for_full_phase3_model_comparison`

Phase3比較への移行には、対象構造について適用可能な未来幅のうち
少なくとも2幅で、P2構造余裕の95%再標本化区間下限が次を満たすことを要求する。

- ROC曲線下面積 > 0.5
- 平均適合率 > 陽性率

正式監査の完了は、Phase3予測器の性能成立を意味しない。
