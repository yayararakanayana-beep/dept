# P1 因果的予測状態系列 実装・検証契約

## 1. 位置づけ

P1は予測器ではない。

固定5軸の正本 `G_t`・`K_t` と、G2/G3で成立した識別子付き関係場基盤から、予測起点ごとに因果的な関係場状態 `X_t` を独立生成する段階である。

```text
X_t = Ψ(G_0:t)
```

時点`t`より後の分布、RF-10未来結果、正解、作用、イベント、Task3/Task4内部状態は使用しない。

## 2. GitHub確認結果

G2（PR #174）とG3（PR #176）は`main`へ統合済みだったが、P1に対応するPR、コミット、ブランチ、ファイルはGitHub上に存在しなかった。

したがって本実装は、`main`のG2/G3/RF-3〜RF-9を正本としてP1を再構成したものである。

## 3. 正式入力

各予測起点`t`について、次を独立生成する。

- G2因果的履歴表示 `G_0:t`
- RF-3: 最新の単一遷移 `G_{t-1} → G_t`
- RF-5: `G_0:t`の時間候補関係場
- RF-6: RF-5候補流れの勾配・循環分解
- RF-7: 分布形状・流路幅
- RF-8: 軸間結合・自己増強・新規駆動・未解決残差
- RF-9: 構造リスク候補・根拠・反証

RF-4はRF-3の科学的監査基盤であり、予測起点ごとの状態ではないため`X_t`へ入れない。RF-10は未来正解側を含むため入力禁止とする。

## 4. 正本形式

`X_t`の正本は固定長ベクトルではなく、識別子付き可変集合である。

各数値配列は次を保持する。

- 安定した特徴識別子
- 出典段階・出典成果物・出典配列
- スコープ種別とスコープ識別子
- 次元役割
- 単位
- 正規化識別子
- 配列形状・型・内容ハッシュ
- 利用可能状態
- 予測起点

G3の識別子正準順を使用し、固定5軸の位置番号だけへ依存しない。

## 5. 終端抽出

長いRF成果物の途中行をそのまま予測特徴として流用しない。

予測起点`t`で独立生成した成果物から、次を抽出する。

- 現在分布 `G_t`
- RF-3最新遷移の全数値
- RF-5/RF-6の最新遷移に属する候補族
- RF-5〜RF-9の最新行
- 時間全体で定義される結合行列など、終端一行へ縮約できない科学量
- RF-7〜RF-9の終端意味成果物

候補族の本数は固定しない。保存候補数を確率として扱わない。

## 6. リスク構造台帳

次の四種類を必須基準として登録する。

1. 過収束
2. 固定化
3. 発散
4. 回復余地低下

ただし四種類だけで閉じない。RF-9に存在する次も保存する。

- 新規駆動併発候補
- 未解決残差優勢候補
- 観測上の戻り抑制
- 勾配／循環優勢修飾
- 成立根拠
- 反証
- 該当軸
- 候補幅・未解決残差

台帳は追加可能であり、単一危険度への合算は禁止する。

## 7. 後退差分

一階後退差分は、直前の予測起点との間で次が一致する数値配列だけに対して計算する。

- 特徴識別子
- スコープ識別子
- 単位
- 正規化
- 形状

```text
D1_t = (X_t - X_previous) / (t - previous_t)
```

二階後退差分は、隣接する二つの一階変化率の差を、その中点間隔で割る。

欠測、候補数変化、スコープ変化、形状変化はゼロで埋めず、利用不能理由として記録する。

## 8. builderと独立validator

builderは予測起点ごとに親成果物と`X_t`を生成する。

validatorは保存済みの合格表示を信用せず、次を再実行する。

- G2構造・履歴表示の検証
- RF-3/RF-5/RF-6/RF-7/RF-8/RF-9の独立validator
- 終端数値配列の再抽出
- 意味成果物の再抽出
- リスク状態の再構成
- 候補記録の再構成
- 一階・二階後退差分の再計算
- マニフェストの完全ファイル集合検証
- 正本・親成果物の非変更確認

manifestを再生成した数値改ざん、未登録ファイル、契約版改ざんも拒否する。

## 10. 科学的主張と限界

成立を主張するもの。

- 因果的接頭部だけから独立した`X_t`を生成できる
- 識別子付き可変集合として保存できる
- 同一過去・異なる未来で同一成果物になる
- 比較可能な成分だけから後退差分を生成できる
- builderと独立validatorで改ざん・未来漏洩・不整合を拒否できる

成立を主張しないもの。

- 予測精度
- 連続前兆座標の妥当性
- 未知構造での同等精度
- 真の不可逆性
- 最適作用
- 安全性
- Task3直接予測との優劣

工学判定は`A_causal_prediction_state_builder_and_independent_validator`、科学的位置づけは`B_limited_causal_relation_field_state_without_prediction_accuracy`とする。

## 11. 完了条件

- builderとvalidatorが同じ4起点以上で合格
- 一階・二階差分が独立再計算と一致
- 決定論的二重生成が一致
- 同一過去・異なる未来が一致
- 系列生成と単独生成の同一起点が一致
- 数値改ざん・未登録ファイル・契約改ざんを拒否
- G2/G3/RF回帰を壊さない
- GitHub Actionsのクリーン環境で再現する
