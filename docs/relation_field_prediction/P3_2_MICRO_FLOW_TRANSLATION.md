# P3-2 共通ミクロ流れ翻訳

## 状態

P3-2は、P3-1で確認した正本G/Kと関係場成果物を、P3-3の予測本体が利用できる共通ミクロ流れ情報へ翻訳する。

この段階では未来流れの予測、未来G生成、精度評価を行わない。

## 入力

必須入力：

- 正本G/K軌道
- RF-2固定格子
- cutoff遷移のRF-3成果物
- cutoffまでのRF-5時間候補関係場

任意入力：

- RF-8軸間結合・同一軸変化・履歴条件付き新規駆動・未解決残差

P1、P2、RF-10、未来状態、truth、疑似現実系内部配列は使用しない。

## RF-3とRF-5の役割分担

RF-3はcutoff遷移の最小費用代表流れを提供する。

RF-5は、複数候補とK_t上の時間整合から次を提供する。

- 代表時間経路
- 候補流れ区間
- 共通流れ
- 軸流量区間
- 軸流量加速度
- 方向反転
- 場の変化候補
- 候補残差

翻訳器はRF-3を単一の真の流れとは扱わない。RF-5代表経路とRF-3代表流れが一致する場合は根拠、一致しない場合は反証として保存する。

## 共通ミクロ流れの単位

固定格子上の`canonical_edge_id`を基本単位とする。

方向は時点ごとに変わるため、次を分ける。

```text
固定位置識別子：fixed-edge:<canonical_edge_id>
有向識別子：edge:<canonical_edge_id>:direction:<+1|-1>
```

反転時も固定位置識別子は維持し、有向識別子と実際の出発・到着セルを反転させる。

平行移動する流れ型そのものの系譜は、RF-5相対記述を使う後続拡張に残す。P3-2で新しい恣意的な系譜IDを作らない。

## 翻訳する情報

各活性固定辺について次を出力する。

- 固定位置流れID
- 現時点の有向流れキー
- 軸
- 正方向基準の始点・終点
- 実際の出発・到着セル
- 現在代表流量
- 直前代表流量
- 流量変化
- 加速度候補
- 活性化・継続・増強・減衰・反転・停止
- 現在候補流量の最小・最大・単純平均
- 候補幅
- 候補内の符号競合
- 最適経路群に共通する最小流量
- RF-3代表流量と方向一致
- 同方向増強・減衰
- 逆方向候補
- 同じセルへ接続する活性辺
- 現在G_t上の移動元利用可能質量
- 軸文脈
- 根拠
- 反証
- 利用不能項目
- 未校正の確度状態

数値が存在しない項目を0で埋めない。

確度は現時点で校正されていないため、次とする。

```json
{
  "confidence": null,
  "confidence_status": "not_calibrated_candidate_evidence_only"
}
```

## 軸文脈

RF-5から次を付与する。

- 軸流量最小・最大・平均
- 軸流量加速度
- 軸方向反転

RF-8が与えられた場合だけ次を追加する。

- 最新時間差の軸間結合区間
- 正・負符号の候補合意
- 同一軸の増強・減衰・反転候補
- 履歴条件付き新規駆動量
- 正規化得点
- 新規駆動候補

RF-8の結合は観測上の関係であり、因果係数とは扱わない。

RF-8がない場合は0を入れず、利用不能理由を保存する。

## 未解決残差

RF-5のcutoff遷移候補について、輸送残差L1の最小・最大・単純平均を保存する。

RF-8がある場合は、RF-8の残差台帳から同じ値も保存する。

履歴条件付き新規駆動と輸送残差は合算しない。

## 活性流れの選択

次のいずれかで非ゼロとなる固定辺を翻訳対象にする。

- RF-5直前代表流れ
- RF-5現在代表流れ
- RF-5現在候補区間
- RF-3cutoff代表流れ

全12,500辺へ空の行を作らず、予測本体に必要な活性辺だけを出力する。

## 成果物

```text
<output>/
  identity.json
  micro_flows.jsonl
  residual_context.json
  summary.json
  validation.json
  provenance.json
```

### micro_flows.jsonl

一行が一つの固定辺上の共通ミクロ流れ情報となる。

### identity.json

- 軌道ID
- cutoff_t
- G_tハッシュ
- K_t履歴鎖ハッシュ
- RF-3・RF-5・任意RF-8の識別子
- 最大読取り時点
- 未来情報未読
- 生状態未読
- 書戻しなし
- 予測未実施

を保存する。

## 実装

```text
scripts/relation_field_prediction_p3_flow_translation.py
```

実行例：

```bash
python scripts/relation_field_prediction_p3_flow_translation.py \
  --gk-trajectory <gk trajectory> \
  --grid-artifact <rf2 grid> \
  --rf3-artifact <rf3 artifact> \
  --rf5-artifact <rf5 artifact> \
  --rf8-artifact <optional rf8 artifact> \
  --cutoff-t 8 \
  --output work/p3_2_translation
```

## 完了境界

P3-2で完了するもの：

- G/K・RF-3・RF-5の時間とハッシュ整列
- RF-8任意接続
- 共通ミクロ流れ情報への翻訳
- 候補幅、反証、残差、利用不能の保持
- 活性辺間の局所隣接関係
- P3-3用成果物生成

P3-2で行わないもの：

- 現在流れの未来継続
- 加速度補正
- 局所結合補正
- 未来G生成
- 予測精度・バランス評価

これらはP3-3以降の範囲である。
