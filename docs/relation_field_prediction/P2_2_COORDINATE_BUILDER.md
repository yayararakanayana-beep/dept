# P2-2 連続座標構築器・独立検証器

## 1. 位置づけ

P2-2は、P1で構築した因果的関係場状態 `X_t` を、RF-9条件面に対する連続座標 `Z_t` へ変換する。

```text
Z_t = C(X_t)
```

これは未来予測器ではない。前兆性、未来発生確率、到達時間、作用推奨は出力しない。座標が実際に未来事象へ先行するかはP2-4で監査する。

## 2. 正式入力境界

入力は検証済みP1成果物だけである。P1の正式入力境界を継承し、RF-10、未来の`G_t/K_t`、正解、シナリオ、seed、split、作用、イベント、世界内部状態、Task3/Task4内部状態を読まない。

P2は親P1マニフェストの完全ファイル集合を検証し、親成果物の前後ハッシュが一致することを確認する。

## 3. 座標台帳

機械可読台帳は64項目を登録する。

- 状態・変化座標
- 予測適用可能性座標
- RF-9原子条件余裕
- 過収束・固定化・発散・回復余地低下の構造余裕
- 勾配／循環修飾
- 将来拡張用の予約座標

四構造は必須基準だが閉集合ではない。単一危険度への合算は禁止する。

経路数、戻り経路、脱出障壁、遷移選択肢、盆地持続性は、経路同一性とゲーム構造が未確立のため予約状態に留める。辺数やセル数を代理値として埋めない。

## 4. 候補区間と条件余裕

各利用可能座標は下限・登録中心・上限を保存する。保存候補数を確率として扱わない。

条件面までの符号付き余裕は、条件側を正とする。

```text
m = orientation * (value - boundary) / comparison_scale
```

- `m < 0`: 条件面の外側
- `m = 0`: 条件面
- `m > 0`: 条件側

論理合成は次に固定する。

- AND: 最小値
- OR: 最大値
- NOT: 符号反転

平均、任意加重和、学習重み、根拠と反証の相殺は行わない。

## 5. 四構造余裕

```text
過収束 = min(形状収束, 流路狭窄, 同方向増強)
固定化 = min(過収束, 観測上の戻り抑制)
発散 = min(形状発散, 流路拡幅, max(同方向減衰, 軸停止))
回復余地低下 = min(回復弱化, -回復強化反証)
```

候補幅、新規駆動、未解決残差は適用可能性情報として分離し、四構造への加点に使用しない。

## 6. 時間差分

同じ座標識別子、対象範囲、単位、正規化、形状だけを比較する。欠測や不一致をゼロで埋めない。

```text
D1_t = (Z_t - Z_previous) / (t - previous_t)
```

二階差分は隣接する一階変化率を中点間隔で差分する。

## 7. 独立検証器

構築器は `_builder.py`、独立検証器は `_independent_validator.py` に分離する。

独立検証器は構築器や数式補助モジュールをimportせず、次を別実装で再計算する。

- 原始座標と候補区間
- 形状・流路・同軸・回復・残差・修飾条件余裕
- 四構造余裕
- 一階・二階後退差分
- 利用可能状態と予約状態
- P1ブール候補との対応記録

否定試験では、構築器側の形状収束式だけを故意に反転させた成果物を作り、独立検証器が不一致として拒否する。この試験により、単なる同一builder再実行ではないことを確認する。

## 8. 主張境界

成立を主張するもの。

- P1因果状態から連続診断座標を決定論的に構築できる
- 64項目の追加可能な台帳を保持できる
- 候補区間と不確実性を保存できる
- 別計算経路の独立検証器が座標、差分、構造余裕を再計算できる
- 構築器の意図的な数式変異、成果物改ざん、未登録ファイル、契約変更を拒否できる

成立を主張しないもの。

- 前兆性
- 予測精度
- 未知構造への一般化
- 真の不可逆性
- 作用推奨
- 安全性証明

工学判定は `A_causal_continuous_coordinate_builder_and_independent_validator`、科学的位置づけは `B_continuous_diagnostic_coordinate_candidates_without_precursor_validation` とする。
