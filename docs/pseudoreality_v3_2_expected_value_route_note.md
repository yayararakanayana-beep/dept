# PseudoReality v3.2 Expected-Value Route Note

## Position

v3.2 is a separate branch from the v3 baseline. It is not an extension of the
v3.1 stress-governance route.

- v3.1 route: stress cost, tolerance, residual/nonproductive stress, human-like governance.
- v3.2 route: expected-value comparison, exploration cost, information retention, gain optimization.

## Core idea

Exploration is not directly commanded. Instead, the model compares:

1. existing-path expected value, and
2. exploration-cost-adjusted expected value.

When the existing short-dominant path no longer improves total gain, its expected
value falls. If information quality, reversibility, exploration room, and retained
information lower exploration cost enough, exploratory routing becomes rational
without a hand-authored exploration flag.

## Short-to-medium route

Short gain is not directly converted into medium payoff. Successful short gain
and useful exploratory flow are first retained as information. Information then:

- lowers exploration cost;
- lowers friction/viscosity modestly;
- supports medium payoff through reusable route knowledge.

This keeps v3.2 in the gain-optimization lineage rather than the stress-cost
lineage.
