# 作用モジュール 上位圧受容分離 RC1 / Task 2-5

日本語名: **圧-作用変換関数のための検証設計**

コードラベル: `ActionModule_UpperPressureReceptionSplit_RC1_Task2_5_ValidationDesign`

## 0. 位置づけ

この文書は、`PressureIntentBundle` から作用候補群を作るための **圧-作用変換関数** を実装する前に、何を検証し、どの情報を集める必要があるかを固定する契約である。

Task 2-5では、まだ変換関数を実装しない。

Task 2-5で固定するのは、次の6つの検証である。

```text
Task 2-5a:
  単独作用→圧対応表の整理

Task 2-5b:
  状態補正係数の検証

Task 2-5c:
  相互作用補正係数の検証

Task 2-5d:
  圧の混合入力検証

Task 2-5e:
  作用強度曲線の検証

Task 2-5f:
  逆変換検証
```

Task 2-5の成果物は、Task 2-6で作る圧-作用変換関数に入れるための、対応率、補正係数、強度曲線、禁止条件、弱化条件、延期条件である。

## 1. 作りたい変換器

最終的に作りたいものは、次である。

```text
入力:
  複数の圧成分

補助入力:
  状態ラベル
  相互作用情報
  作用強度情報

出力:
  作用候補群
```

より具体的には、次の形である。

```text
圧 × 状態 × 相互作用 × 強度
↓
作用候補群
```

ここでの出力は、最終作用ではない。

出力はあくまで、同時発火門、境界guard、ActionFrame生成前に渡す **候補** である。

## 2. 圧と作用を混同しない固定

Task 2-5では、次を厳密に分ける。

```text
圧:
  上位側が下位に求める方向。
  例: 探索頻度を上げたい、巻き戻し感度を上げたい、圧上限を下げたい。

作用:
  疑似現実または実系に与える実際の操作候補。
  例: 探索を増やす作用、余裕枠を増やす作用、揺れを抑える作用。

結果:
  作用を打った後に系で観測される変化。
  例: 戻しやすさが上がる、揺れが減る、副作用が増える。
```

圧-作用対応は、次の順で作る。

```text
作用を打つ
↓
結果を見る
↓
その結果がどの圧成分の目的に合うかを測る
↓
作用→圧対応表を作る
↓
運用時には逆向きに使い、圧→作用候補を作る
```

したがって、Task 2-5で知りたいのは、単なる名前対応ではない。

知りたいのは、次である。

```text
ある作用を打ったとき、
どの圧成分の目的にどれだけ寄与するか。

その寄与率が、
状態、相互作用、強度によってどれだけ変わるか。
```

## 3. 日本語固定名

Task 2-5以降、説明では次の日本語名を主に使う。

コード上の英語名は既存実装との互換のために残すが、説明では日本語名を優先する。

| 日本語名 | コード名 | 意味 |
|---|---|---|
| 探索を増やす作用 | `exploration_injection` | 未知方向・候補生成・試行を増やす作用 |
| 結合を緩める作用 | `coupling_relief` | 強すぎる結びつきや拘束を緩める作用 |
| 揺れを抑える作用 | `volatility_damping` | 変動、振動、過剰な揺れを抑える作用 |
| 不確実性を探る作用 | `uncertainty_probe` | すぐ大きく動かず、不確実な箇所を探る作用 |
| 関係固定を外す作用 | `relation_unlock` | 固定された関係やロックを外す作用 |
| 余裕枠を増やす作用 | `buffer_increase` | 戻しやすさ、安全余地、保留余地を増やす作用 |
| 無作用 | `no_op` | 比較基準。作用しない |

## 4. Task 2-5a: 単独作用→圧対応表の整理

### 4.1 目的

単独作用検証の結果を整理し、各作用がどの圧成分の目的にどれだけ合うかを明示する。

これは、Task 2-6の基本対応表の材料になる。

### 4.2 入力

```text

作用強度
状態ラベル
seed
no_op比較
作用後の観測結果
```

### 4.3 測定するもの

```text
戻しやすさの変化
探索性の変化
揺れの変化
隠れ損傷の変化
疲労の変化
資源偏りの変化
費用の変化
信頼/安定の変化
```

### 4.4 出力表

```text

component_direction
basic_alignment_score
basic_alignment_rank
side_effect_burden
confidence
calibration_status
primary_or_supporting_action
state_sensitivity_flag
interaction_sensitivity_flag
```

### 4.5 完了条件

```text
各作用について、対応する圧成分の上位候補が出ている。
各対応が主作用か補助作用か分かる。
状態依存が強い対応には flag が立つ。
相互作用依存が強い対応には flag が立つ。
```

## 5. Task 2-5b: 状態補正係数の検証

### 5.1 目的

同じ圧入力でも、状態によって作用候補を強めるべきか、弱めるべきか、延期すべきか、禁止すべきかを調べる。

ここは、後で下位レイヤー情報と接続するために特に丁寧に検証する。

Task 2-5bでは、状態を判定する機械は作らない。

今作るのは、状態ラベルを受け取って変換関数に反映するための窓口と、どの状態がどの作用に影響するかの対応である。

### 5.2 状態ラベルの初期集合

状態ラベルは、単なる通常/高負荷だけでは足りない。

下位レイヤー接続を見据えて、最低限、次の状態軸を検証対象にする。

```text
揺れの強さ
余裕の少なさ
固定の強さ
探索過多
探索不足
不確実性の高さ
残差の大きさ
関係の詰まり
隠れ損傷の疑い
巻き戻し可能性
```

### 5.3 状態軸の意味

#### 揺れの強さ

作用を入れた時に、系が振動、発散、過反応へ向かいやすいかを見る。

影響を受けやすい作用:

```text
探索を増やす作用
関係固定を外す作用
結合を緩める作用
```

#### 余裕の少なさ

失敗時に戻せる余地があるかを見る。

影響を受けやすい作用:

```text
関係固定を外す作用
探索を増やす作用
結合を緩める作用
```

余裕が少ない場合、これらは弱化、延期、禁止の対象になり得る。

#### 固定の強さ

関係、履歴、経路が固まりすぎているかを見る。

影響を受けやすい作用:

```text
結合を緩める作用
関係固定を外す作用
探索を増やす作用
```

固定が強い場合は解除系候補を残す意味があるが、揺れや余裕不足と同時に出る場合は危険化する。

#### 探索過多

すでに探索や候補生成が多すぎるかを見る。

影響を受けやすい作用:

```text
探索を増やす作用
不確実性を探る作用
```

探索過多の場合、探索を増やす作用は弱化または延期されるべきである。

#### 探索不足

停滞、過固定、未探索領域の不足があるかを見る。

影響を受けやすい作用:

```text
探索を増やす作用
不確実性を探る作用
結合を緩める作用
```

探索不足の場合、探索系作用は強める候補になる。

#### 不確実性の高さ

見えていない部分が多いか、観測信頼が低いかを見る。

影響を受けやすい作用:

```text
不確実性を探る作用
余裕枠を増やす作用
揺れを抑える作用
```

不確実性が高い場合、大きく動かす作用より、観測・余裕・抑制に寄せる。

#### 残差の大きさ

既存観測で説明できないズレが大きいかを見る。

影響を受けやすい作用:

```text
不確実性を探る作用
探索を増やす作用
余裕枠を増やす作用
```

残差が大きい場合、いきなり強い作用を打つより、探査と余裕確保が必要になる。

#### 関係の詰まり

特定の関係、経路、結合が詰まっているかを見る。

影響を受けやすい作用:

```text
結合を緩める作用
関係固定を外す作用
不確実性を探る作用
```

関係の詰まりが強い場合、解除系や緩和系の候補が出る。ただし、揺れや余裕不足との同時確認が必要である。

#### 隠れ損傷の疑い

表面上の改善に見えても、内部負荷が増えていそうかを見る。

影響を受けやすい作用:

```text
探索を増やす作用
関係固定を外す作用
結合を緩める作用
```

隠れ損傷の疑いが強い場合、これらは弱化、延期、または余裕枠を増やす作用との併用対象になる。

#### 巻き戻し可能性

作用後に戻せるかを見る。

影響を受けやすい作用:

```text
関係固定を外す作用
探索を増やす作用
結合を緩める作用
```

巻き戻し可能性が低い場合、不可逆に近い作用は候補から落とすか、強く弱化する。

## 6. Task 2-5c: 相互作用補正係数の検証

### 6.1 目的

複数作用が同時に候補化された場合に、足し算で扱ってよいか、相殺するか、有用に増幅するか、副作用が増幅するかを検証する。

Task 2-5では、関係性と組み合わせを **相互作用補正** としてまとめて扱う。

ただし監査列では、関係由来なのか、作用同士の組み合わせ由来なのかを分けて残す。

### 6.2 検証対象

最低限、次の組み合わせを見る。

```text
探索を増やす作用 + 関係固定を外す作用
余裕枠を増やす作用 + 揺れを抑える作用
結合を緩める作用 + 関係固定を外す作用
探索を増やす作用 + 揺れを抑える作用
不確実性を探る作用 + 結合を緩める作用
```

### 6.3 出力表

```text
action_channel_a
action_channel_b
interaction_type
interaction_modifier
useful_amplification_score
cancellation_score
side_effect_amplification_score
block_combination_flag
dampen_combination_flag
defer_combination_flag
interaction_reason
relation_dependency_flag

```

### 6.4 完了条件

```text

相殺する組み合わせが明示されている。
```

## 7. Task 2-5d: 圧の混合入力検証

### 7.1 目的

### 7.2 検証単位

```text

pressure_strength_vector
```

### 7.3 見るべき圧混合の型

```text

探索を増やしたい圧 + 固定を下げたい圧
固定を上げたい圧 + 圧上限を下げたい圧
固定を下げたい圧 + 採用ハードルを下げたい圧
診断深度を上げたい圧 + 不感帯幅を下げたい圧
```

## 9. Task 2-5f: 逆変換検証

### 9.1 目的

圧を作用候補群へ変換し、その作用候補群を疑似系に適用した結果が、元の圧の意図に戻して見たときに合っているかを検証する。

これは、圧-作用変換関数を作る前の最終検証である。

### 9.2 流れ

```text
圧入力
↓
作用候補群を生成
↓
疑似系に作用
↓
結果を観測
↓
結果を圧成分方向に戻して評価
↓
元の圧意図と一致していたかを測る
```

### 9.3 出力表

```text
pressure_input_id
pressure_components
candidate_action_set
applied_action_set
result_trace
reconstructed_pressure_effect
intent_alignment_score
side_effect_burden
recoverability_score
exploration_preservation_score
overfixation_risk
divergence_risk
rollback_required
```

### 9.4 完了条件

```text
圧入力の意図と結果が対応しているか分かる。
副作用が見える。
戻しやすさが見える。
過固定化、過発散、探索喪失が見える。
```

## 11. Task 2-6で作る関数の予定形

Task 2-6で作る関数は、概念的には次の形になる。

```text
作用候補スコア
=
圧の強さ
× 基本対応率
× 状態補正
× 相互作用補正
× 校正信頼度
− 副作用ペナルティ
```

ただし、Task 2-5ではこの関数をまだ実装しない。

Task 2-5では、上記の各項に入れる値を検証で集める。

## 12. 禁止事項

Task 2-5では、次を禁止する。

```text
圧から最終Actionを直接決める
状態補正なしで関係固定を外す作用を強く採用する
探索を増やす作用と関係固定を外す作用を無条件に同時採用する
状態判定器を下位レイヤー接続前に勝手に作る
下位レイヤーから来ていない状態情報を実観測済みとして扱う
未校正の対応を校正済みとして扱う
```

## 13. 既定の保守動作

下位レイヤーの状態信号がまだ接続されていない場合、次の保守動作を使う。

```text
状態不明:
  強い解除系作用は弱化
  探索を増やす作用は上限を低くする
  余裕枠を増やす作用を優先候補に残す
  揺れを抑える作用を安全側候補に残す
  相互作用が危険な組み合わせは延期または弱化
```

## 15. Task 2-5の結論

Task 2-5の結論は次である。

```text
圧-作用変換関数は、固定の圧→作用表だけでは作れない。

必要なのは、
単独作用→圧対応、
状態補正、
相互作用補正、
圧混合入力、
作用強度曲線、
逆変換検証
を先にそろえることである。

状態は今は仮ラベルで受け取る。
後で下位レイヤー情報と接続する。

関係性と組み合わせは相互作用補正としてまとめる。
ただし監査では由来を分ける。
```
