# Task 3.1f-4.1 構造数横断・構造群安定性追加監査

## 位置づけ

Task 3.1f-4の正式判定は変更しない。Task 3.1f-4.1は、個別構造の一対一対応だけでは捉えられない分割・統合の揺れを、複数構造の非負合成による「構造群」として追加監査する。

本タスクは構造数を自動選択しない。`selection_lock.json`を生成せず、比較推薦だけを出力する。

- Stage B/C Artifact ID: `8245413464`
- Artifact digest: `sha256:77570fa06b2a3b50571b6480515d95afe6882878992a767aa738e67b02cce51b`
- 使用分割: fit 1,082行、validation 256行

## 比較対象

主要比較構造数は次で固定する。

```text
5 / 8 / 10 / 15
```

構造数間の全比較は次の6組とする。

```text
5→8 / 5→10 / 5→15 / 8→10 / 8→15 / 10→15
```

## ネイティブ5軸署名

各候補構造は3,125セルを`(5,5,5,5,5)`へC-orderで再形成し、次を算出する。

1. 資源余裕・情報品質・圧力・探索余地・可逆性の各5区分周辺分布
2. 各軸の平均区分、分散、エントロピー
3. fit全体の重み付き平均分布との差に基づく軸コントラスト
4. 全10組の二軸同時分布
5. 相互情報量、正規化相互情報量、符号付き区分相関、独立分布との差

軸寄与比率は識別上の説明指標であり、因果的な成分割合とは扱わない。

## 構造群対応

低構造数側の1構造を、高構造数側の最大4構造による非負凸結合で近似する。

- 最大群サイズ: 4
- 構造群成立: JS類似度 0.85以上
- 中間領域: 0.80以上0.85未満
- 有効メンバー最小重み: 0.10
- 小さい群から探索し、閾値を満たす最小群を優先
- 群探索は基底だけで行い、validation活性度を群選択へ使用しない

分類は`strong_single`、`group_preserved`、`group_rescued`、`intermediate`、`unresolved`とする。

## 活性度整合性

構造群を固定した後、fitだけで非負回帰係数を求め、validationへ固定適用する。validationで群構成または係数を再調整しない。

評価指標はPearson相関、Spearman相関、コサイン類似度、正規化平均絶対誤差、p95絶対誤差、活性上位10％一致率とする。

## 安定性監査

### 初期値安定性

各構造数の代表runと収束済みrunを比較する。未収束runは正式分母へ含めない。比較runが3未満の場合は不合格ではなく`evidence_limited`とする。

### 80％部分集合安定性

既存契約の決定論的・グループ保持80％部分集合5種類を変更せず再使用する。

- 構造数5: 既存部分集合基底を再利用
- 構造数8・10・15: 同じ設定で追加再学習
- solver: multiplicative update
- loss: generalized KL
- initialization: nndsvda
- max_iter: 2,000
- tolerance: 1e-5
- regularization: none

部分集合基底を固定し、validationは係数推定だけに使用する。

## 独立監査

解析側は`comparative_recommendation_candidate.json`だけを生成する。独立監査側は、元Artifactと保存された部分集合モデルから全派生表を再計算し、完全一致を確認した場合だけ`comparative_recommendation.json`を生成する。

独立監査は次も確認する。

- 入力Artifact ID・digest・主要ファイルハッシュ
- holdout不在
- `selection_lock.json`不在
- 基底・活性度・部分集合モデルのハッシュ
- 全出力Artifact manifest
- 自動構造数選択が行われていないこと
