# Task 3.1f-1 比較設計

## 0. 目的

Task 3.1eで構築した1,582件×3,125セルの完全質量分布から、人間が意味論軸を先に与えず、複数条件で繰り返し現れる非負構造を抽出するための比較方法を固定する。

本設計が決めるのは、**構造抽出方法と安定性の判定方法**である。構造の意味や最終Core軸数は決めない。

---

## 1. 主解析行列

### 1.1 行列定義

主解析行列を `X` とする。

- 行：Task 3.1eの各分布スナップショット
- 列：3,125セル
- 値：各セルの質量
- 形状：`(1582, 3125)`
- 型：`float64`
- 各行：有限・非負・総和1

`mass_matrix.npy` の行は、`snapshot_metadata.csv` と `discovery_manifest.csv` の `matrix_row_index` によって対応付ける。

行順だけに依存した結合は禁止し、以下を独立に確認する。

- 行数一致
- `matrix_row_index` が0から連続
- `snapshot_id` 一致
- `dataset_split` 一致

### 1.2 分割

既存のTask 3.1e分割をそのまま使用する。

| 分割 | 行数 | 用途 |
|---|---:|---|
| fit | 1,082 | 構造基底の学習、初期値安定性、データ摂動安定性 |
| validation | 256 | 構造数の選択、代表runの補助判定、選択規則の適用 |
| holdout | 244 | 選択固定後の一度限りの最終確認 |

行単位のランダム再分割は禁止する。

同一外部ベクトル、同一source run、同一対応関係から生じた行が別分割へ混入するような再分割も禁止する。

### 1.3 主入力に含めない情報

以下はTask 3.1fの主モデル入力へ加えない。

- 外部要因6値
- 外部要因名
- 22地形情報
- seed
- source step
- 通常分布／外部要因作用後分布のラベル
- matched pair情報

これらは評価、層別集計、説明、監査にのみ使用する。

## 2. 前処理

### 2.1 非負質量の保持

非負行列分解の入力には、Task 3.1eの質量をそのまま使用する。

禁止する処理：

- 平均中心化
- 負値を生む標準化
- 対数変換
- セルごとの恣意的な重み付け
- 小さい質量の切り捨て
- 外部要因値の混入
- 通常分布と外部要因作用後分布の事前分離学習

Jensen–Shannon距離計算時の数値保護に限り、固定値 `1e-12` を用いる。入力行列そのものは書き換えない。

### 2.2 分析重み

`discovery_manifest.csv` の `analysis_weight` を正式な標本重みとする。

主手法の一般化KL損失では、KLダイバージェンスの斉次性を利用し、学習入力の各行を次のように拡大する。

```text
X_fit_weighted[i, :] = analysis_weight[i] * X_fit[i, :]
```

これは、各行のKL損失を `analysis_weight[i]` 倍することに対応する。

重みを二重適用してはならない。

正式判定では次を両方出力する。

- 重み付き集計
- 重みなし集計

主選択指標には重み付き検証指標を使用し、重みなし指標は偏り監査に使用する。

### 2.3 基底の確率分布化

非負行列分解後、各構造基底行の総和を1へ正規化し、その倍率を対応する活性度列へ移す。

```text
H'[j, :] = H[j, :] / sum(H[j, :])
W'[:, j] = W[:, j] * sum(H[j, :])
```

この変換は再構成 `W @ H` を変えない。

正規化後の各基底行を、3,125セル上の構造分布として扱う。

構造基底行の総和が0、非有限、または負値を含む場合、そのrunは失敗とする。

### 2.4 活性度比率

各スナップショットの構造活性度比率を次で定義する。

```text
activation_share[i, j] = W'[i, j] / sum(W'[i, :])
```

行総和が0または非有限の場合、そのrunは失敗とする。

## 3. 比較する方法

## 3.1 本線：重み付き一般化KL非負行列分解

正式な構造抽出方法は以下で固定する。

- 方法：非負行列分解
- solver：multiplicative update
- beta loss：Kullback–Leibler
- 正則化：なし
- `l1_ratio = 0`
- 最大反復回数：2,000
- 収束許容値：`1e-5`
- 構造数：固定grid
- 学習入力：重み付きfit行列
- validation／holdout変換：固定した基底に対する非負係数推定

実装ライブラリのバージョンはrun manifestへ必ず記録する。同じ数理契約を満たす限り、ライブラリのパッチ版差はタスク変更としない。

### 3.1.1 初期値

各構造数について次を実行する。

#### 決定論的基準run

- 初期化：`nndsvda`
- 1 run

#### ランダム初期値run

- 初期化：`random`
- seed：
  - `31011`
  - `31012`
  - `31013`
  - `31014`
  - `31015`
  - `31016`

ランダム初期値runは6本で固定する。

少なくとも5本が収束しなければ、その構造数は収束条件を満たさない。

決定論的基準runは構造比較には使用するが、6本中5本という収束率の分母には含めない。

## 3.2 感度参照：Frobenius非負行列分解

主手法の選択後、選ばれた構造数と、その固定grid上の直前・直後の構造数だけを対象に実行する。

例：15が選択された場合、12・15・20を実行する。

- solver：coordinate descentまたは同等のFrobenius最小化
- loss：Frobenius
- 重み付け：各fit行を `sqrt(analysis_weight)` 倍する
- seed：`31201`, `31202`, `31203`
- 最大反復回数：2,000
- 許容値：`1e-5`

これは主構造数の選択には使用しない。KL損失固有の構造になっていないかを確認する感度監査である。

## 3.3 線形参照：重み付き主成分分析

同じ構造数gridで重み付き主成分分析を実行する。

- fitの重み付き平均を計算
- `sqrt(analysis_weight)` を掛けた中心化行列へ特異値分解を適用
- fit以外にはfit平均とfit主成分だけを適用

主成分は負値を持ち得るため、意味論構造候補にはしない。

出力を二種類に分ける。

1. 生の線形再構成
   - RMSE
   - MAE
   - 負値率
   - 行総和誤差
2. 確率単体へユークリッド射影した再構成
   - Jensen–Shannon距離
   - 総変動距離

単純なclip＋再正規化ではなく、確率単体への決定論的ユークリッド射影を使用する。

主成分分析は線形低次元表現の参照であり、Task 3.1fの採用モデルにはしない。

## 3.4 平均分布基準

fitの重み付き平均分布を全行へ適用するrank-0基準を必須とする。

選択された非負行列分解は、validationおよびholdoutでこの基準より改善しなければならない。

## 3.5 辞書学習

辞書学習はTask 3.1fの自動実行対象に含めない。

非負行列分解が安定性条件を満たさず、局所・疎構造の別手法が必要になった場合だけ、理由と追加契約案を提示してユーザー確認を得る。

確認なしに辞書学習へ切り替えることは禁止する。

## 3.6 クラスタリング

クラスタリングは構造抽出方法として使用しない。

選択された活性度空間に明確な群があるかをTask 3.1f-5で補助確認できるが、クラスタ番号を構造軸やCore軸とはみなさない。

## 5. 再構成評価

## 5.1 非負行列分解の再構成

生の再構成を次で定義する。

```text
X_hat_raw = W' @ H'
```

分布距離用の再構成は、各行を総和1へ正規化する。

```text
X_hat_dist[i, :] = X_hat_raw[i, :] / sum(X_hat_raw[i, :])
```

この正規化で行総和誤差を隠してはならない。生の行総和誤差を別指標として必ず保存する。

## 5.2 必須指標

各run、各split、各層別群について次を計算する。

- 平均二乗平方根誤差
- 平均絶対誤差
- Jensen–Shannon距離の平均・中央値・p95・最大
- 総変動距離の平均・中央値・p95・最大
- 生再構成の行総和絶対誤差
- 非有限値数
- 負値数

層別群：

- `dataset_split`
- 通常分布／外部要因作用後分布
- `source_step`
- `active_factor_count`
- `vector_origin`

全指標について重み付き・重みなしを分ける。

## 5.3 外部要因変形の保持

`matched_pairs.csv` を使用し、実分布と再構成分布について次を計算する。

- 外部分布と対応通常分布のJensen–Shannon距離
- 上記距離の絶対保存誤差
- signed deformationのコサイン類似度
- signed deformationの相対L1誤差

signed deformationは評価にのみ使用する。

```text
actual_delta = X_external - X_base
reconstructed_delta = Xhat_external - Xhat_base
```

この差分行列を別の因子モデルへ入力することはTask 3.1fの範囲外である。

## 6. 構造対応付けと安定性

## 6.1 構造間距離

総和1へ正規化した構造基底同士について、平方根Jensen–Shannon距離を使用する。

自然対数を使った平方根Jensen–Shannon距離の最大値は `sqrt(log(2))` である。

正規化類似度を次で定義する。

```text
js_similarity = 1 - js_distance / sqrt(log(2))
```

範囲は0から1とする。

コサイン類似度も補助指標として保存するが、run間対応付けの主費用はJensen–Shannon距離とする。

## 6.2 run間対応付け

同一構造数のrun間では、Hungarian法で構造を一対一対応付けし、Jensen–Shannon距離の合計を最小化する。

構造番号の一致を安定性とみなしてはならない。

## 6.3 代表run

各構造数で収束したrunのうち、他runとの対応後平均 `js_similarity` が最大のrunを代表runとする。

これは構造集合のmedoidに相当する。

同値の場合のみ、validationの重み付き平均Jensen–Shannon距離が小さいrunを選ぶ。それでも同値の場合はseed番号が小さいrunを選ぶ。決定論的基準runはseed番号を `0` として扱う。

最低validation誤差だけでrunを選ぶことは禁止する。

## 6.4 初期値安定性条件

構造数が初期値安定性を満たす条件：

1. ランダム6runのうち5run以上が収束する。
2. 収束run間の対応構造類似度中央値が `0.85` 以上。
3. 代表runの各構造について、類似度 `0.80` 以上で対応するrunの割合を生存率とする。
4. 代表構造の80%以上が、生存率 `0.75` 以上を満たす。

これらはTask 3.1fの運用判定値であり、普遍的な数学定数とは主張しない。

## 6.5 grouped 80%摂動

選択された構造数について、fitデータの80%を使用する決定論的部分集合を5個作る。

外部要因作用後分布は `external_vector_id` を単位としてまとめて採否を決める。通常分布は `source_run_id` を単位とする。

外部要因作用後グループは次で層別する。

- `active_factor_count`
- `vector_origin`

各層内で固定saltとグループIDのSHA-256値を使って決定論的に80%を選ぶ。

salt：

- `task3_1f_subset_01`
- `task3_1f_subset_02`
- `task3_1f_subset_03`
- `task3_1f_subset_04`
- `task3_1f_subset_05`

各部分集合は決定論的基準初期化1runで学習し、全fit代表runへ対応付ける。

摂動安定性条件：

- 対応類似度中央値 `0.80` 以上
- 代表構造の80%以上が、5部分集合中4以上で類似度 `0.75` 以上

## 6.6 world seed感度

fitのworld seed `0` のみ、world seed `1` のみで、選択構造数の決定論的基準runを1回ずつ学習する。

全fit代表runへ対応付け、結果を保存する。

これは強い診断項目であるが、半数データとなるため単独では失敗判定にしない。中央値が `0.75` 未満の場合はTask 3.1gへの引き渡しを `conditional` とする。

## 6.7 step別評価

step別に新しい基底を学習しない。

選択済み基底を各stepへ適用し、再構成誤差と活性度分布を層別評価する。

独立した時間構造抽出はK_t側の課題であり、Task 3.1fへ混ぜない。

## 7. 重複構造と未使用構造

## 7.1 重複構造

代表run内の構造対について、次のどちらかを満たす場合を重複対とする。

- `js_similarity >= 0.98`
- `cosine_similarity >= 0.995`

少なくとも1つの重複対へ含まれる構造の割合を重複構造率とする。

構造数の適格条件：

```text
重複構造率 <= 0.20
```

## 7.2 未使用構造

fitにおける `activation_share` の重み付き95パーセンタイルが `0.005` 未満の構造を未使用構造とする。

構造数の適格条件：

```text
未使用構造率 <= 0.20
```

稀だが特定条件で強く活性化する構造は未使用扱いにしない。95パーセンタイルと最大値、活性化群を一緒に保存する。

---

## 8. 構造数の適格条件

各構造数は、次をすべて満たした場合に `admissible` とする。

1. 入力・出力完全性検査に合格。
2. ランダムrun収束数が5/6以上。
3. 初期値安定性条件に合格。
4. 重複構造率が0.20以下。
5. 未使用構造率が0.20以下。
6. validation再構成が平均分布基準より良い。
7. validationの全層別群で非有限値・負値・空群がない。
8. 外部要因作用後分布だけを極端に悪化させていない。

最後の条件は、外部要因作用後分布の重み付き平均Jensen–Shannon距離が、通常分布の値の3倍を超えた場合に不合格とする。

これは通常分布の少数行だけをよく再構成し、主な外部要因分布を無視する抜け道を防ぐためである。

---

## 9. validationだけによる構造数選択

holdoutを一切使用せず、次の順序で選択する。

1. `admissible` な構造数だけを残す。
2. 各構造数について、収束runごとのvalidation重み付き平均Jensen–Shannon距離を集計する。
3. 平均値が最小の構造数を `best_error_rank` とする。
4. `best_error_rank` のrun間標準誤差を計算する。
5. 次を満たす最小の構造数を選択する。

```text
rank_mean_validation_js <= best_error_mean + best_error_standard_error
```

これはone-standard-error ruleであり、誤差が実質同等なら小さい構造数を選ぶ。

admissibleな構造数が0の場合、Task 3.1fの主仮説は未支持と判定し、holdoutを開かない。

選択後、以下を `selection_lock.json` へ固定する。

- 選択構造数
- 代表run ID
- 基底ファイルSHA-256
- 活性度ファイルSHA-256
- contract SHA-256
- Task 3.1e入力artifact SHA-256
- admissible rank一覧
- validation選択指標
- 選択規則の再計算結果

## 10. holdout一度限り評価

holdoutは `selection_lock.json` 作成後に別工程で初めて読み込む。

holdoutでは選択済み非負行列分解、平均分布基準、同一rankの主成分分析参照だけを評価する。

holdoutを見て別rankへ変更してはならない。

### 10.1 判定

#### confirmed

次をすべて満たす。

- holdout重み付き平均JS距離が平均分布基準より5%以上改善
- holdout平均JS / validation平均JS `<= 1.25`
- holdout p95 JS / validation p95 JS `<= 1.35`
- 完全性・構造・対応関係の独立検査に全合格

#### conditional

次をすべて満たす。

- holdout重み付き平均JS距離が平均分布基準より改善
- holdout平均JS / validation平均JS `<= 1.50`
- holdout p95 JS / validation p95 JS `<= 1.50`
- 完全性検査に全合格

#### failed

上記以外。

`failed` の場合、別rankへの自動切替や閾値変更は禁止する。原因を整理してユーザー確認を得た後、新しい契約版を作る。

## 11. Task 3.1gへ渡す構造

Task 3.1gへ渡せる構造は、選択代表runのうち次を満たすものとする。

- 初期値生存率0.75以上
- grouped 80%摂動で5中4以上生存
- 重複対へ含まれない、または重複理由が明示されている
- 未使用構造ではない
- validationで有限な活性度を持つ

world seed感度が低い構造は削除せず、`conditional_structure` として渡す。

Task 3.1fでは意味論名を付けず、IDを `S001`, `S002`, ... とする。
