# Task 3.2-4.1 Rev1 実装契約

## 目的

Task 4.1bで確定した三つの修正範囲を一体で実装する。

1. 構造正解の再定義
2. 分岐継続仮定の分離
3. 作用強度・開始遅延の適応的境界探索

加えて、改訂した構造正解が時点や現在リスクだけではなく、Task 3・Task 4の情報から識別可能かを軽量予測器で検証する。

## 構造正解

- 構造縮小
- 危険な縮小
- 不可逆化進行
- 環境依存悪化
- 内在的回復能力喪失
- 実現環境下・安定基準環境下の脱出観測
- 脱出観測時だけの条件付き脱出費用
- 回復後再悪化率
- 枝先状態の幾何的広がり
- 回復作用種類の多様性
- 安全な到達可能範囲

## 継続仮定

- C0: 現在入力を8ステップ固定後に中立化。旧版互換監査のみ。
- C1: 分岐直後から中立化。外乱解除感度のみ。
- C2: 元軌道の未来入力を再生。実現環境側の主要正解。
- C3: 同seed安定継続の未来入力を再生。内在構造側の主要正解。

四条件は平均化しない。

## 分岐予算

- 粗い探索: 3,600分岐
- 境界細分化: 最大1,080分岐
- 確率境界追加複製: 最大400分岐
- 絶対上限: 5,100分岐

## 簡易予測検証

比較候補:

- 現在リスク・現在価値
- Task 3特徴
- Task 4特徴
- Task 3＋Task 4特徴

二値対象には固定ロジスティック回帰、連続対象には固定Ridge回帰を用いる。validationで候補を固定し、selection lock検証後だけholdout状態を開く。snapshot stepは正式特徴へ入れず、時点だけの基準を独立監査する。

## 正式seed

- fit: 30, 31, 32
- validation: 33
- holdout: 34

## 禁止事項

- Action Module接続
- Parameter Box書込み
- 元軌道・G/K/O_tへの書戻し
- scenario、seed、split、snapshot step、未来結果、構造正解の特徴混入
- 脱出未観測への便宜的な脱出費用代入
- 強い作用ほど良いという単調性仮定
- holdout確認後の候補再選択
- 普遍的不可逆性や実運用性能の主張
