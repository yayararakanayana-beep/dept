# Task 3.2-4.1 範囲
## 構造的リスク・縮小均衡・脱出可能性検証

## 1. 位置づけ

Task 3.2-4.1は、Task 3.2-4の内部追加検証である。

上位の6タスク構成は変更しない。

```text
Task 3.2-3
近距離の危険と強度

Task 3.2-4
遠距離の危険方向

Task 3.2-4.1
安全な将来へまだ移動できるか
```

## 2. 目的

現在値や運動が安定している場合でも、次が悪化していないかを測る。

- 最小脱出費用
- 回復成功率
- 到達可能価値
- 到達可能範囲
- 最終作用可能時間
- 再固定化率
- 必要同時作用規模の代理

現在実現価値の低下ではなく、将来可能性の縮小を構造的リスクとして扱う。

## 3. 分岐試験

保存済みPseudoReality v3.3状態を新しい世界へ複製し、元軌道とは独立した反実仮想分岐を作る。

各判定時点から次を試す。

- 無作用
- 資源・需要緩和
- 競争・制約緩和
- 情報ノイズ・急性外乱緩和
- 上記の複合作用
- 弱・中・強の作用
- 即時・4ステップ遅延・8ステップ遅延
- 作用終了後の無作用追跡

分岐世界は元軌道へ書き戻さない。

## 4. 構造正解

分岐結果から、判定時点ごとに次を求める。

```text
minimum_escape_cost
best_recovery_probability
reachable_value
reachable_range
last_action_window
refixation_probability
minimum_simultaneous_action_scale
provisional_irreversibility_level
shrinking_equilibrium_candidate
```

これらは未来側の正解であり、予測入力へ入れない。

## 5. 暫定不可逆性段階

```text
0：無作用で回復
1：弱い狭い作用で回復
2：中程度の作用で回復
3：強い作用または広い同時作用で回復
4：試した作用集合では回復不能
```

これは今回の作用集合・予算・時間幅に対する条件付き評価である。

絶対的不可逆性ではない。

## 6. 縮小均衡

現在価値の変化が小さい一方で、次のうち複数が悪化する場合を縮小均衡候補とする。

- 最小脱出費用が上昇
- 到達可能価値が低下
- 到達可能範囲が縮小
- 最終作用可能時間が短縮
- 均衡維持費用が上昇

現行PseudoReality v3.3でこの構造が発生しない場合、正例を人工的に作らない。

その場合は世界モデルまたは作用集合が構造リスク検証に不足していると判定する。

## 7. 安全領域

安全領域はfit分割の`stable_continuation`だけから校正する。

次を使う。

- リスク値
- 現在実現価値
- 損傷
- 負の生存圧
- 経路支持
- 回復速度
- 生存余力

安全領域への一時到達だけでは回復成功としない。

一定期間の持続と、作用解除後の追跡を要求する。

## 8. 現在実現価値

Task 4.1内の暫定観測値として、次を合成する。

```text
正の要素：
net_viability_value
route_support
viability_reserve
expected_value_advantage
effective_medium_payoff
recovery_speed

負の要素：
maintenance_cost
operating_cost
exploration_cost
negative_viability_pressure
```

この合成は最終価値関数として固定しない。

## 9. 必要同時作用規模

PseudoReality v3.3には独立した複数プレイヤーがいない。

そのため、同時に変更する外部入力軸数を協調規模の代理として使う。

これはプレイヤー人数ではない。

## 10. 軽量予測器

重い分岐試験で作った構造正解を、現在までの観測だけから予測できるか確認する。

比較候補：

1. 現在リスクのみ
2. Task 3の現在状態・履歴
3. Task 4の分布マクロ力学
4. Task 3＋Task 4

Task 4の固定候補は次を使う。

```text
distribution__D08__dmdc__macro_only__H16
```

## 11. データ分割

Task 4.1専用seedを使う。

```text
fit：20, 21, 22
validation：23
holdout：24
```

同じ元軌道から派生した全分岐は元軌道の分割を継承する。

分岐単位の無作為分割は禁止する。

## 12. Holdout隔離

validationまでで次を固定する。

- 安全領域
- 作用集合
- 作用費用
- 構造正解定義
- 使用特徴
- 予測器候補
- 選択予測器

selection lockが独立検証されるまで、holdout状態ファイルを開かず、holdout分岐を実行しない。

## 13. 成功判定

### A

- 構造正解に十分な変動がある
- 縮小均衡候補が観測される
- 複数の不可逆性段階が出る
- 選択予測器が現在リスクだけの基準を明確に超える

### B

- 脱出費用や不可逆性段階には変動がある
- 一部の構造量は予測可能
- 縮小均衡検出または一般化はまだ弱い

### C

- 構造正解に有効な変動がない
- 縮小均衡が発生しない
- 作用集合で回復境界を識別できない
- 現在リスクだけと差がない

## 14. 範囲外

- 正式なゲーム構造
- 実プレイヤー数
- 正式な協調人数
- 絶対的不可逆性の証明
- 実世界への一般化
- Action Module接続
- Parameter Box更新
- 正式G_t・K_t
- 動的関係場の固定

## 15. 完了条件

- 状態を新しい世界へ分岐複製できる
- 元軌道へ書き戻さない
- 強度・遅延・作用解除後追跡を実行できる
- 最小脱出費用を求められる
- 回復成功率を求められる
- 到達可能価値・範囲を求められる
- 最終作用可能時間を求められる
- 再固定化率を求められる
- 暫定不可逆性段階を求められる
- 縮小均衡を監査できる
- Task 3・4特徴から軽量予測できる
- selection lock後にholdout評価できる
- A・B・C判定を出せる
