# Task 3.2-3 → Task 3.2-4 引き渡し

## 1. Task 4の目的

Task 3.2-4では、生状態と履歴からマクロ力学表現を粗く抽出し、それがTask 3.2-3の単純基準を超えるか確認する。

Task 4は、最終的なマクロ力学モデルを固定する段階ではない。

## 2. 超えるべき正式基準

```text
history_logistic__W08__H04
```

- 履歴幅: 8
- 予測時間幅: 4
- 警報閾値: 0.2
- 警報継続条件: 1
- selection lock hash: `e733d440c194fa60b4e7f1ffd0555629c5807572c2c1fc770783298bc735a32c`

## 3. Task 3 holdout性能

- 高リスク軌道検出率: 1.0
- 警報先行時間中央値: 4ステップ
- 誤警報率: 0
- 適合率・再現率曲線下面積: 1.0
- リスク深度順位相関: 1.0
- リスク深度平均絶対誤差: 0.000967

現行データでは分類・順位性能に上限余地がほとんどない。

したがってTask 4は、単に同じデータで正解率1.0を再現するだけでは上乗せ効果を示せない。

## 4. Task 4の改善対象

優先順位は次とする。

### 第一優先

- 4ステップより早い警報
- 未知の外部入力予定での性能維持
- 外乱開始時刻・強度を変えた場合の安定性

### 第二優先

- リスク深度数値誤差の縮小
- 固定化候補と崩壊候補の構造的区別
- リスク方向・回復方向の説明可能性

### 第三優先

- 観測されていない力学変化を残差として検出
- 不確実性上昇時の棄権・警戒

## 5. 比較条件

Task 4とTask 3を公平に比較するため、次を揃える。

- 同じfit / validation / holdout分割
- 同じ評価指標
- 同じ未来情報禁止
- 同じscenario ID・seed入力禁止
- 同じ警報成立の評価方法
- Task 3 selection lockを変更しない

Task 4のモデル選択はTask 4独自のvalidationで行ってよいが、Task 3基準を再調整してはならない。

## 6. 現行データの限界

現行探索コーパスでは、各seedに対して同じ外部入力予定を繰り返している。

そのため、Task 4開始時には少なくとも追加の難化検証を設ける。

候補:

1. 外乱開始時刻をずらす
2. 外乱終了時刻をずらす
3. 外力強度を連続的に変える
4. 複数外力の組合せ比率を変える
5. 通常→悪化→部分回復→再悪化を作る
6. 固定化と崩壊が同じ現在リスク点数を持つ局面を作る

この難化データは、Task 3の正式selection lockを変更するためには使わない。

## 7. マクロ力学候補の扱い

次は現時点で仮説候補にすぎない。

```text
状態変化
=
地形に沿った流れ
+
循環・非勾配流
+
粘性・応答遅延
+
拡散・確率的広がり
+
外力・外部流入出
+
未解決成分
```

Task 4ではこの分解を最初から固定しない。

複数の既存手法を粗く試し、予測・説明・残差保持に寄与する要素だけを残す。

## 8. Task 4で避けること

- Task 3より複雑だから有効とみなす
- 同じ現在状態特徴を別名で再投入するだけのモデル
- scenario予定を暗記するモデル
- 全残差を既知力学へ強制吸収する
- 正解ラベルを使って力学成分を後付けする
- holdoutを見て手法を切り替える

## 9. Task 4の成功条件

最低限、次のどれか1つをTask 3基準より明確に改善する。

- 未知条件での高リスク検出率
- 未知条件での警報先行時間
- 誤警報率
- リスク深度数値誤差
- 固定化・崩壊の区別
- seedおよび外力予定変更に対する安定性

改善が確認できない場合、マクロ力学抽出は現段階では有望でない、またはデータ設計が不十分と判定する。
