# Task 3.2-3 Rev1 Phase 2：パイロットコーパス成立性監査

## 1. この段階の目的

Phase 2の目的は、予測器を作ることではない。

正式な予測検証へ進む前に、次の三点が成立するかを確認する。

1. 必要な結果群を、生の疑似現実軌道から実際に観測できるか。
2. 生成側が保存したシナリオ名や正解ラベルを信用せず、生状態から結果を独立再計算できるか。
3. 結果の出現率とばらつきから、正式検証に必要な独立軌道数を見積もれるか。

ここでいう独立標本は軌道である。同一軌道から切り出した複数の時点窓は、独立標本数へ加えない。

## 2. 全体の流れ

```text
Phase 1で固定した設計契約
↓
分布外条件を持つ小規模な試験軌道を生成
↓
各軌道の生状態配列を保存
↓
生状態配列だけからリスク推移を再計算
↓
同じ環境条件の安定参照軌道との差を計算
↓
8種類の結果群を独立判定
↓
結果出現率・ばらつき・必要軌道数を監査
↓
正式コーパスへ進めるか判定
```

## 3. 正解側の独立性

### 3.1 使用するもの

各時点の`states/step_*.npz`に保存された次の生配列だけを使用する。

- 分布
- 損傷
- 固定性
- 摩擦
- 粘性
- 回復速度
- 経路支持
- 生存余力
- 負の生存圧
- 直前移動量

これらから各時点のリスク値、集中度、移動量、固定性を再計算する。

### 3.2 結果固定前に使用しないもの

- `summary.json`
- `truth.jsonl`
- `metrics.jsonl`
- シナリオ名
- 予定していた結果群
- Task 4のマクロ特徴
- Task 4.1の反実仮想正解

独立再計算結果をハッシュで固定した後に限り、既存の暫定ラベルとの一致・不一致を比較用診断として読む。

既存ラベルとの一致率は、独立再計算結果を変更しない。

## 4. 安定参照軌道

各試験軌道は、次が同一の安定参照軌道と比較する。

- 分割
- seed
- 背景状態
- 世界パラメーター構成

異なるのは追加された外乱系列だけである。

この安定参照は正解計算側だけで使用する。後続の予測器の入力へは入れない。

## 5. 試験条件

### 5.1 分割

- fit
- validation
- holdout

ただし、これは正式なモデル評価用分割ではない。Phase 2では三分割すべてを成立性監査のために確認できる。

Phase 2で生成した軌道は、後続の正式fit・validation・holdoutへ再利用しない。正式コーパスでは、schedule、背景状態、世界パラメーター構成、seedをすべて新しくする。

### 5.2 分布外条件

分割間で次を重複させない。

- 具体的な外乱系列（8類型の名称自体は共通語彙として維持する）
- 背景状態
- 世界パラメーター構成
- seed

seedだけを変えて同じ外乱系列を再利用する方式は、分布外検証として認めない。

### 5.3 背景状態

次を含む。

- ほぼゼロの安定背景
- 非ゼロの安全参照
- 分布中心を緩やかに動かす背景
- 通常変動幅が異なる背景

fitでは同じ外乱系列を異なる背景状態・世界パラメーター構成へ置き、結果が背景依存で変わるかも監査する。

### 5.4 世界パラメーター構成

既存の疑似現実の数式は変更しない。既存の設定窓口から、次の範囲を変える。

- 移動量
- 拡散量
- 自然回復率
- 基本回復速度
- 生存余力
- 経路支持
- 経路支持喪失率

結果を作るために状態配列を直接書き換えることは禁止する。

## 6. 監査する結果群

生状態から、次の8種類を独立判定する。

1. 安全継続
2. 完全回復
3. 部分回復
4. 遅延回復
5. 再悪化
6. 持続悪化
7. 固定化候補
8. 崩壊候補

どの条件にも十分当てはまらない場合は、既知の結果へ強制せず`unresolved`として残す。

## 7. 難しい正例・負例

### 7.1 難しい負例

- 強いが回復可能
- 一時的な高リスクだが持続しない
- 現在状態が同じで未来が反対
- 悪化傾向の後に回復
- ノイズを含む安全継続

### 7.2 難しい正例

- ゆっくり蓄積する悪化
- 外乱除去後に遅れて危険化
- 回復後の再悪化
- 現在は安全だが履歴に前兆がある
- 急激な外部衝撃を伴わない固定化

## 8. 同じ現在状態・異なる未来

完全回復系列と持続悪化系列は、所定の分岐時点まで次を完全に同一にする。

- seed
- 背景状態
- 世界パラメーター構成
- 外乱系列

分岐時点の全状態配列を内容ハッシュで比較し、同一であることを確認する。

その後の未来系列だけを、回復側と悪化側へ分ける。

```text
同じ現在状態
├─ 外乱を除去する未来
└─ 悪化入力を継続する未来
```

状態同一性は必須の実装検証である。一方、二つの未来が本当に異なる結果群へ分かれるかは、疑似現実系の表現力を測る科学的結果であり、強制しない。

## 9. 出現率・ばらつき・必要件数

軌道単位で次を保存する。

- 結果群別の出現数と出現率
- 95%ウィルソン区間
- 最大相対リスク深度の分散
- 軌道単位再標本化による95%区間
- 予測起点8・16・24、未来幅4・8・16ごとの陽性・陰性軌道数
- 必要陽性20軌道・必要陰性20軌道を得るための保守的な推奨軌道数

出現率が0または1に張り付き、必要件数を推定できない場合は、便宜値を入れず未推定とする。

## 10. Phase 2の判定

### 成立

次をすべて満たした場合だけ、正式コーパス設計へ進める。

- 生状態だけから結果を再計算できる
- 分布外条件が分割間で分離されている
- 8種類の結果群が必要最低数だけ観測される
- 難しい正例・負例の設計が揃っている
- 同じ現在状態の内容同一性が成立する
- 同じ現在状態から異なる未来結果が観測される
- 全起点・未来幅で必要件数を推定できる

### 不成立

一つでも不足する場合は、正式予測へ進まず、どの結果群・背景条件・未来幅が不足したかを保存する。

Phase 2のプログラムが正常完了することと、科学的に正式コーパスへ進めることは分けて判定する。監査が正常完了しても、科学的判定は不成立になり得る。

## 11. この段階で行わないこと

- 正式コーパス生成
- 予測特徴量生成
- 予測器実装
- 学習
- 候補選択
- 選択固定
- 正式validation・holdout評価
- Task 4特徴の使用
- Task 4.1正解の使用
- 正式G_t・関係場の構築
- ゲーム構造分類
- 不可逆性の証明
- 作用選択

## 12. 次段階

Phase 2が成立した場合、次に正式コーパス規模と生成契約を固定する。

成立しなかった場合は、結果を作為的に付け替えず、不足した軌道条件だけを再設計してPhase 2を再監査する。
