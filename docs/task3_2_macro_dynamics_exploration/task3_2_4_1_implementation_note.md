# Task 3.2-4.1 実装ノート

## 実装方針

Task 4.1は、既存の保存済み状態を新しいPseudoReality v3.3世界へ複製し、外部入力窓口だけを使って反実仮想分岐を実行する。

元軌道、Parameter Box、G/K、Action Moduleへは書き戻さない。

## 分岐規模

正式実行では次を使う。

- 30元軌道
- 各軌道3判定時点
- 無作用1種
- 回復作用4種
- 作用強度3段階
- 開始遅延3段階
- 各条件2複製

合計予定分岐数：

```text
30 × 3 × (1 + 4 × 3 × 3) × 2
= 6,660分岐
```

## 予測比較

- 現在リスクのみ
- Task 3特徴
- Task 4特徴
- Task 3＋Task 4特徴

validationで候補を固定し、selection lock検証後にholdout状態を読み、holdout分岐を実行する。

## 結果の読み方

結果がAまたはBなら、現行世界と作用集合に構造的な脱出難度の差があり、観測から軽量予測できる可能性がある。

結果がCでも、正例を人工的に作らず、次のいずれかを切り分ける。

- 現行世界に縮小均衡機構が不足
- 試した作用集合が狭い
- 安全領域定義が厳しすぎる
- 判定時点が粗い
- 観測特徴から構造正解を予測できない
