# Task 3.2-1 範囲固定
## 生状態・履歴ログ・高リスク評価基盤

## 1. タスク種別

- 新規タスク: Task 3.2-1
- 親フェーズ: Task 3.2 マクロ力学探索
- 基準ブランチ: `main`
- 性質: データ契約・入力境界・採点境界の実装

## 2. 中心目的

次の予測関係を検証可能にする。

```text
現在の生状態 X_t
+
現在までの履歴ログ L_t
        ↓
将来の高リスク構造
不可逆性
リスク深度
作用可能な時間幅
```

完全な次状態再現は補助評価であり、主目的ではない。

## 3. 固定事項

- 予測入力は`X_t + L_t`とする。
- 生状態・生入力・生イベントの時系列を正本とする。
- 派生特徴は正本ログから再計算可能にする。
- 現在状態と履歴は別の入力領域として保持する。
- モデル入力、採点用正解、再現用情報を分離する。
- 将来状態、将来入力、将来イベント、結果ラベルをモデル入力へ入れない。
- データ分割は軌道単位とし、隣接行の無作為分割を禁止する。
- 高リスク判断を主目的とし、通常時の全変動再現を必須にしない。
- 不可逆性、リスク深度、作用可能時間を採点可能な欄として保持する。

## 4. 非固定事項

次は検証前に固定しない。

- マクロ力学の構成要素
- 地形、流れ、循環、粘性、拡散、外力等の分解
- 履歴幅
- モード数
- 抽出手法
- 高リスク閾値
- 安全領域境界
- 不可逆性の数式
- リスク深度の数式
- 予測時間幅
- 固定5軸への正式な射影方法

## 5. データ領域

### モデル入力

時点`t`までに観測済みの情報だけを含む。

- 現在状態参照
- 過去状態参照
- 過去および時点`t`で既知の外部入力
- 観測済みイベント
- 実施済み作用
- 観測済み応答

### 採点用正解

モデルには渡さない。

- 次状態
- 将来のリスク事象
- リスク発生までの時間
- 回復結果
- 回復時間
- 不可逆性段階
- リスク深度候補
- 不可逆境界通過時刻

### 再現用情報

- trajectory ID
- scenario ID
- seed
- 初期状態ID
- world/config/code版
- dataset split

## 6. 時間順序

各ステップは`pre_transition`状態として記録する。

```text
X_tとL_tを記録
↓
時点tで既知の入力・イベントを記録
↓
世界を1ステップ進める
↓
X_{t+1}を別状態として保存
↓
採点用正解を更新
```

状態レコードは`0..total_steps`、次状態正解は`0→1`から`total_steps-1→total_steps`までを持つ。

## 7. 不可逆性の暫定段階

- 0: 自然回復可能
- 1: 弱い作用で回復可能
- 2: 中程度以上の作用が必要
- 3: 強い作用でも回復が不安定
- 4: 試験範囲では回復不能

これは採点欄の順序尺度であり、正式な不可逆性理論ではない。未評価時は`null`とする。

## 8. リスク深度の候補

単一指標へ統合せず、次を別々に保持する。

- 安全状態への復帰時間
- 必要作用強度
- 回復確率
- 危険状態の持続時間
- 悪化速度
- 安全基準からの距離
- 利用可能な回復経路数
- 再悪化確率

## 9. 実装

- 契約: `configs/task3_2_1_macro_dynamics_contract.json`

- 高リスク結果を後から採点できる。

- 契約テストが通る。
