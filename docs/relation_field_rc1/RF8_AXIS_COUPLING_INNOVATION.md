# 固定5軸 関係場 RF-8 — 軸間結合・自己増強・新規駆動・未解決残差

## 1. 目的

Task RF-8は、RF-5〜RF-7で得られた時間候補関係場から、次の四種類を分けて記録する。

```text
軸間結合候補
自己増強・自己減衰候補
履歴条件付き新規駆動候補
未解決輸送残差
```

RF-8で扱う結合は観測上の関係であり、因果関係ではない。

また、履歴で説明しにくい新規駆動候補を、外部入力・外部ショック・外因とは呼ばない。

---

## 2. 入力境界

RF-8は次を読む。

- 正本`gt_mass.npy`
- 正本`history_ledger.csv`
- RF-2固定格子
- RF-5時間候補関係場
- RF-6勾配・循環分解
- RF-7分布形状・流路幅

RF-5の`start_t`から`to_t`までだけを使用する。

最低3遷移を必要とする。

```text
3遷移
→ 2個の時間差結合
```

を作れるためである。

未来のG_t、履歴接尾部、外部入力、作用、イベント、正解情報は読まない。

---

## 3. 候補軸流れ族

RF-5の保存最適経路群から、各遷移で実際に使われる候補番号を集める。

同じ候補番号が複数の保存経路に現れても、一度だけ数える。

各候補について、RF-2の辺向きに沿った軸別符号付き流量を求める。

```text
軸aの符号付き流量
=
軸aに属する全辺の正味流量の和
```

各遷移について、

- 最小値
- 最大値
- 重複除去後の単純平均
- 候補幅

を保存する。

格子の一辺は0.25なので、座標変位換算も保存する。

```text
座標変位
=
符号付き軸流量 × 0.25
```

候補平均は期待値ではない。

---

## 4. 位置条件付き軸結合

関係場の軸結合は、ある軸上の位置によって、別軸方向の流れがどう変わるかとして読む。

RF-8では、元フレームの重心軸`a`と、同じ遷移の符号付き流量軸`b`について、二変量のリッジ傾きを計算する。

```text
位置条件付き結合[a,b]
=
重心軸aから流量軸bへの二変量傾き
```

5×5行列として保存する。

候補流れに幅があるため、平均傾きだけでなく、流量区間を線形伝播した傾き最小値・最大値も保存する。

### 読み方の制限

この傾きは、

- 因果効果ではない
- 他軸を統制した偏効果ではない
- 普遍的な関係場係数ではない

局所履歴上の位置依存結合候補である。

元軸の位置変化が小さすぎる場合は識別不能として保存する。

---

## 5. 時間差軸結合

連続する二遷移について、現在流量と次時点の流量加速度を組み合わせる。

```text
流量加速度
=
次遷移の流量
-
現在遷移の流量

時間差結合[a,b]
=
現在の軸a流量
×
軸b流量加速度
```

これは5×5行列になる。

候補流れ族の全組合せについて計算し、

- 最小値
- 最大値
- 単純平均
- 正符号で全候補一致
- 負符号で全候補一致

を保存する。

正符号は「軸aの現在方向と軸bの加速方向が一致した」候補であり、因果的促進を意味しない。

負符号も因果的抑制を意味しない。

---

## 6. 同一軸の時間変化

同じ軸について、現在流量と次流量を比較する。

### 同方向増強候補

```text
符号が同じ
＋
次流量の絶対値が増える
```

### 同方向減衰候補

```text
符号が同じ
＋
次流量の絶対値が減る
```

### 方向反転候補

```text
現在流量 × 次流量 < 0
```

### 軸活性化候補

```text
現在はほぼ0
＋
次時点で非0
```

### 軸停止候補

```text
現在は非0
＋
次時点でほぼ0
```

候補流れ族の全組合せで成立した場合だけ、合意ラベルを立てる。

候補組合せの成立割合も保存するが、確率とは呼ばない。

### 意味上の制限

同方向増強候補は、引力の自己増強を証明しない。

同方向減衰候補も、引力の消失や発散を証明しない。

RF-9で形状変化・境界持続・流路幅・可逆性候補と統合するための部品である。

## 7. 履歴条件付き新規駆動

現在の流れが過去の関係場からどれだけ外れたかを、過去接頭部だけから計算する。

### 7.1 過去基準

現在遷移`t`の元重心と、過去遷移の元重心の距離を求める。

近い状態ほど大きく、古い遷移ほど小さく重み付けする。

```text
状態重み
=
exp(-距離² / (2 × 0.35²))

時間重み
=
0.8 ^ 経過遷移数
```

両者を掛けて正規化し、過去軸流量の重み付き平均を履歴基準とする。

### 7.2 新規駆動量

```text
履歴条件付き新規駆動量
=
現在の候補平均流量
-
過去履歴基準流量
```

過去変動、現在候補幅、数値下限を合わせた尺度で正規化する。

### 7.3 診断ラベル

次を満たす軸へ候補ラベルを立てる。

- 過去遷移数が2以上
- 有効履歴支持が1.5以上
- 正規化得点が2以上
- 絶対差が`1×10⁻⁶`以上

ラベル名は、

```text
履歴条件付き新規駆動候補
```

である。

### 呼んではならないもの

```text
外部入力
外部ショック
外因
真の構造転換
原因
```

とは呼ばない。

同じ観測は、

- 内部関係場の変化
- 未観測状態
- モデル不足
- 非局所遷移
- 観測誤差
- 外部由来の影響

のどれでも起こり得る。

## 8. 未解決輸送残差

RF-5の輸送逆算では、

```text
観測分布差
=
局所辺流れによる再構成
＋
輸送残差
```

となる。

RF-8は、最適経路群で使われる候補の輸送残差について、

- L1量
- 正残差質量
- 負残差質量
- 元重心に対する5軸残差モーメント

の最小・最大・平均を保存する。

### 新規駆動との分離

```text
輸送残差
=
現在の局所辺輸送でも説明できない分布差

履歴条件付き新規駆動
=
現在の流れが過去接頭部基準から外れた量
```

であり、別物である。

局所輸送として完全に再構成できても、過去と異なる流れなら新規駆動候補になり得る。

逆に、過去と同じ傾向でも、局所辺輸送で表せない分布差なら輸送残差が残る。

両者を合算しない。

## 10. 合格条件

- 保存経路の重複数を候補確率として使わない
- 候補流れ最小・最大・平均・幅を保存
- 合成位置依存流れの二変量傾きを復元
- 同方向増強・減衰・反転・活性化・停止を区別
- 過去にない軸切替えを履歴条件付き新規駆動候補として検出
- 新規駆動を外部要因と呼ばない
- 履歴新規駆動とRF-5輸送残差を別成果物で保存
- RF-5・RF-6・RF-7・正本履歴の同一性を確認
- RF-5の`to_t`より後を読まない
- 同一入力から同一成果物
- 親成果物と正本を書き換えない
- マニフェスト改変を検出

## 11. 科学的位置づけ

RF-8の位置づけは、

```text
B_limited_axis_coupling_and_history_innovation
```

である。

成立したのは、固定5軸・有限履歴上で、

- 位置条件付き二変量結合
- 時間差結合
- 同一軸の増強・減衰候補
- 過去接頭部基準からの逸脱
- 局所輸送残差

を分離して保存する基盤である。

未確定なのは、

- 因果方向
- 長期安定性
- 真の引力・排斥
- 新規駆動の由来
- 不可逆性
- リスク判定

である。

## 12. RF-9への引き渡し

RF-9では、RF-6〜RF-8を統合してリスク構造候補を作る。

中心となる組合せは、

```text
同方向増強
＋
分布収縮・集中
＋
流路細化
＋
境界からの回復流れ低下
```

による過収束・固定化候補と、

```text
同方向減衰・軸停止
＋
分布拡張・分散
＋
流路拡幅
```

による発散候補である。

さらに、履歴条件付き新規駆動候補と輸送残差を、既存リスク構造とは別の不確実性として保持する。

RF-9でも直ちに未来予測精度を主張せず、まず構造検出の整合性を検証する。
