# 固定5軸 関係場 RF-2 — 格子接続構造

## 1. 目的

Task RF-2は、固定5軸の3,125セルを、後続の局所流れ逆算で使用する決定論的な疎グラフへ変換する。

本タスクはG_t・K_tを読み取らない。流れ、関係場、予測、リスク構造も推定しない。

## 2. セル識別子

軸順はRF-1契約から変更しない。

1. 資源余裕
2. 情報品質
3. 圧力
4. 探索余地
5. 可逆性

各軸5区分で、セル座標は5個の区分番号として表す。

```text
(i0, i1, i2, i3, i4),  0 <= ik <= 4
```

セル番号はC順で固定する。

```text
cell_id = ((((i0 * 5 + i1) * 5 + i2) * 5 + i3) * 5 + i4)
```

軸ごとのセル番号の歩幅は次である。

```text
[625, 125, 25, 5, 1]
```

セル番号は0から3,124までであり、座標との往復変換が完全に一致しなければならない。

## 3. 隣接規則

二つのセルが隣接する条件は、次の両方を満たす場合だけである。

- 一つの軸だけが異なる
- その軸の区分番号の差が1である

斜め接続、二軸同時接続、飛越し接続、端から端への循環接続は作らない。

正本辺は、区分の低いセルから高いセルへ向けた正方向だけを1本保存する。逆方向は近傍構造から派生させる。

```text
基準辺数       12,500本
双方向近傍数   25,000本
```

各辺の位相距離は1、固定5軸座標上の距離は0.25である。

この接続は意味、シナリオ、正解情報、観測された流れによって変更しない。

## 4. 境界と次数

各セルについて、軸ごとの下限境界と上限境界をビットマスクで保持する。

各軸の内部区分では正負2方向、境界区分では内向き1方向だけを持つ。そのためセル次数は5から10になる。

| 次数 | セル数 |
|---:|---:|
| 5 | 32 |
| 6 | 240 |
| 7 | 720 |
| 8 | 1,080 |
| 9 | 810 |
| 10 | 243 |

## 5. 疎な接続演算子

### 5.1 近傍構造

双方向近傍はCSR形式で保存する。

- `indptr`
- `indices`
- `edge_ids`
- `axis`
- `direction`
- `coordinate_distance`

現在セルから区分の高い側へ進む近傍方向を`+1`、低い側を`-1`とする。

3,125×3,125の密な隣接行列は保存しない。

### 5.2 接続演算子

接続演算子は3,125行×12,500列の疎なCOO形式で保存する。

基準辺eがセルsからセルrへ向くとき、列eは次になる。

```text
B[s, e] = -1
B[r, e] = +1
```

非負の基準辺流量`f`に対する分布差分は、

```text
delta_mass = B @ f
```

である。各列の総和は0なので、接続演算子だけで作る変化は総質量を保存する。

発散演算子は`-B`として派生できるため、別の密行列として保存しない。

## 6. 成果物

```text
<output>/
  contract.json
  metadata.json
  nodes.csv
  edges.csv
  nodes.npz
  neighbors.npz
  incidence.npz
  validation.json
  manifest.json
```

- `nodes.csv`: セル番号、5軸座標、境界、次数
- `edges.csv`: 基準辺番号、両端セル、移動軸、距離
- `nodes.npz`: RF-3向けの数値セル配列
- `neighbors.npz`: 双方向CSR近傍
- `incidence.npz`: COO接続演算子と辺配列
- `validation.json`: RF-2検証結果
- `manifest.json`: SHA-256による完全性台帳

NPZは固定ZIP時刻と固定キー順で書き、同じ契約から同一バイト列を再生成できるようにする。

## 7. 実行

生成:

```bash
python scripts/relation_field_grid_rc1.py build \
  --output artifacts/relation_field_grid_rc1
```

検証:

```bash
python scripts/relation_field_grid_rc1.py validate \
  --input artifacts/relation_field_grid_rc1
```

## 8. RF-2の合格条件

- 固定5軸契約と完全一致
- セル番号と5軸座標の往復一致
- 3,125セル
- 基準辺12,500本
- 双方向近傍25,000本
- 全辺が一軸一区分だけの移動
- CSR近傍が対称
- 接続演算子の向きが固定
- 全セルが一つの連結成分に属する
- 境界次数分布が理論値と一致
- 同一入力から同一成果物を再生成
- マニフェスト検証通過
- G_t・K_tを読み取らない

## 9. 非対象

RF-2では次を行わない。

- G_tからG_{t+1}への流れ逆算
- 辺流量の推定
- 関係場の時間整合
- 勾配・循環分解
- 新規駆動・残差推定
- 未来予測
- リスク構造判定
- 作用生成

これらのうち単一遷移流れ逆算はRF-3、流れ監査はRF-4で扱う。
