# 固定5軸 関係場 RF-10 — 構造リスク候補の未来区間予測検証

## 1. 目的

RF-10は、RF-9で作った構造リスク候補が、その時点より後に観測される分布変化をどの程度先取りできるかを検証する。

評価対象は次の4組である。

```text
過収束候補
→ 将来の集中進行

固定化候補
→ 将来の集中持続

発散候補
→ 将来の分散進行

回復余地低下候補
→ 将来の回復失敗
```

RF-10は作用を選ばない。

また、真の不可逆性や因果関係を証明しない。

---

## 2. 最も重要な時間境界

一つのRF-9成果物には、履歴区間の複数時点に対応する行がある。

しかしRF-5以降の時間候補関係場は、成果物の終了時点までの履歴を使っている。

そのため、終了時点より前のRF-9行を未来予測として評価すると、その行より後の履歴情報が間接的に混ざる可能性がある。

RF-10ではこれを禁止する。

```text
一つのRF-9成果物
→ 最終行だけを予測サンプルにする
```

複数時点を評価する場合は、各時点で履歴を切り、別々のRF-5〜RF-9成果物を作る。

```text
時点10の予測
→ to_t = 10 のRF-9成果物

時点20の予測
→ to_t = 20 のRF-9成果物
```

これにより、予測時点より後のG_tを関係場推定へ混ぜない。

---

## 3. 予測側と正解側の分離

RF-10は処理順を固定する。

```text
1. RF-9と親成果物を再検証
2. RF-9最終行だけを予測スナップショットとして凍結
3. 予測スナップショットのハッシュを確定
4. その後で初めて未来G_tを読む
5. 未来結果ラベルを生成
6. 予測と結果を比較
```

未来接尾部は、

```text
正解側
```

だけで使う。

予測値、基準予測、候補定義、結果閾値の変更には使わない。

---

## 4. ケース定義

RF-10はケース一覧JSONを入力する。

```json
{
  "manifest_version": "relation_field_predictive_validation_case_manifest_rc1",
  "cases": [
    {
      "case_id": "case_001",
      "partition": "test",
      "trajectory_dir": "...",
      "grid_artifact_dir": "...",
      "rf5_artifact_dir": "...",
      "rf6_artifact_dir": "...",
      "rf7_artifact_dir": "...",
      "rf8_artifact_dir": "...",
      "rf9_artifact_dir": "..."
    }
  ]
}
```

一つのケースは、

```text
一つの履歴終了時点
一つのRF-9最終行
一つの未来接尾部
```

に対応する。

分割は次の3種類である。

```text
development
validation
test
```

同じ軌道識別子を複数分割へ置くことは禁止する。

---

## 5. 評価する未来幅

初期設定は次の3種類である。

```text
1遷移後
2遷移後
4遷移後
```

RF-9終了時点を`t`とすると、結果側は、

```text
G_t
G_{t+1}
...
G_{t+h}
```

だけを使う。

---

## 6. 未来結果の基礎量

各未来フレームから次を測る。

```text
正規化エントロピー
有効支持数
最大セル質量
総分散
境界質量
```

これらは未来結果の評価用であり、RF-9予測側へ戻さない。

---

## 7. 将来の集中進行

終了時点と比べて未来端点で、次の4信号のうち3つ以上が成立した場合に立てる。

```text
正規化エントロピーが0.02以上低下
有効支持数が0.85倍以下
最大セル質量が0.05以上増加
総分散が0.005以上低下
```

これは観測上の集中進行であり、吸引域や過収束の真実ラベルではない。

---

## 8. 将来の集中持続

未来端点で集中進行が成立し、さらに未来幅の半数以上のフレームで集中進行条件が成立した場合に立てる。

```text
端点集中
かつ
未来フレームの50%以上で集中
```

固定化候補の評価対象として使う。

真の不可逆性は意味しない。

---

## 9. 将来の分散進行

終了時点と比べて未来端点で、次の4信号のうち3つ以上が成立した場合に立てる。

```text
正規化エントロピーが0.02以上増加
有効支持数が1.15倍以上
最大セル質量が0.05以上低下
総分散が0.005以上増加
```

発散候補の評価対象として使う。

---

## 10. 将来の回復失敗

2遷移以上の未来幅だけで評価する。

まず未来端点より前に、次のどちらかの事象が必要である。

```text
集中信号4個中3個以上
または
境界質量が終了時点より0.25以上増加
```

その事象から未来端点までに、次の回復信号を数える。

```text
正規化エントロピーが0.02以上回復
有効支持数が1.15倍以上回復
最大セル質量が0.05以上低下
境界質量が0.20以上低下
```

回復信号が2個未満なら、回復失敗結果を立てる。

事象がなければ評価対象外とする。

---

## 11. 基準予測

RF-9候補は次の二つと比較する。

### 常に候補なし

```text
全サンプルでFalse
```

### 単一要素基準

```text
過収束候補の基準
= 形状収束だけ

固定化候補の基準
= 形状収束 AND 境界回復弱化

発散候補の基準
= 形状発散だけ

回復余地低下候補の基準
= 境界回復弱化だけ
```

これにより、RF-9の複合構造が単純な一要素判定より有用かを比較できる。

---

## 12. 評価指標

各分割・未来幅・候補種別ごとに次を出す。

```text
真陽性
偽陽性
真陰性
偽陰性
適合率
再現率
特異度
正解率
均衡正解率
F1
マシューズ相関係数
ブライア得点
```

適合率と再現率には95%ウィルソン区間を付ける。

分母が0の指標は0にせず、未定義として保存する。

---

## 13. 新規駆動と未解決残差

次の二つは初期RF-10では主予測対象にしない。

```text
新規駆動併発候補
未解決残差優勢候補
```

代わりに、これらがある群とない群で未来結果率を分けて保存する。

これは因果効果や上昇効果の推定ではない。

---

## 14. 精度合格を禁止する条件

小規模な合成検証が通っても、予測精度が担保されたとは判定しない。

精度主張には最低限、

```text
未知testケース30件以上
かつ
各候補・各未来幅で陽性5件以上
かつ
各候補・各未来幅で陰性5件以上
```

が必要である。

この条件を満たさない場合、RF-10の判定は、

```text
評価器は動作した
予測精度は未判定
```

に留める。

## 16. 合格条件

- RF-9最終行だけを使う
- 予測凍結後に未来接尾部を読む
- 正本履歴の終了時点ハッシュがRF-9と一致
- 同じ軌道を複数分割へ置かない
- 合成例で集中・分散・回復失敗・回復成功を区別
- 指標計算を既知の混同行列で再現
- 未来接尾部変更で予測は不変、結果だけ変化
- 同一入力から同一成果物
- 親成果物と正本を書き換えない
- マニフェスト改変を検出

---

## 17. 科学的位置づけ

RF-10の位置づけは、

```text
B_limited_leakage_safe_predictive_validation_harness
```

である。

成立したのは、時間漏洩を抑えた予測評価器である。

十分な未知テスト件数が揃うまでは、

```text
リスク構造を高精度に予測できる
```

とは主張しない。
