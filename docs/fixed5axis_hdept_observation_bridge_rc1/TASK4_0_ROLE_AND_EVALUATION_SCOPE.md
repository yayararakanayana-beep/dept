# 固定5軸上位観測翻訳層 RC1 — Task 4-0 役割境界と評価対象の再固定

## 1. この文書の目的

Task 4-0では、Task 4の科学検証を作り直す前に、H-DEPT上位観測層が何を読む装置であり、何を読む装置ではないかを固定する。

ここではコード、47特徴、H11対応表、Task 2生成器、Task 3独立検証器を変更しない。数値閾値や新しい合格条件もまだ決めない。

固定するのは、次の三点である。

1. H-DEPTの観測対象
2. 下位観測層との責任分担
3. 新しいTask 4で許される検証と、禁止する検証

---

## 2. H-DEPTが読む対象

H-DEPTが正式に受け取るものは、下位レイヤーが構成した系全体の分布状態 `G_t` と、その因果的履歴 `K_t` である。

```text
対象系
  ↓
下位観測・関係場構成
  ↓
系全体の分布状態 G_t と履歴 K_t
  ↓
H-DEPT上位観測
  ↓
系全体の性格 M_t / H11_STRUCTURED
```

H-DEPTの中心的な問いは、次である。

> 下位の評価制度全体は、現在どのような分布的・幾何的・時間的性格を示しているか。

H-DEPTは、局所的な現象を直接探索して名前を付ける装置ではない。下位で構成された系全体の状態を、上位の関心軸から再表現する装置である。

---

## 3. 下位レイヤーとの責任分担

### 下位観測・動的関係場が担当するもの

- 局所固定化
- 局所探索
- 局所吸引域
- 局所的な流れ、粘度、障壁、外力
- 局所異常
- 局所的な分裂、統合、回転、振動
- 詳細な地形と脱出経路

これらは、局所単位、関係構造、地形、履歴を詳しく読む下位レイヤーの責任である。

### `G_t / K_t`が保持するもの

下位レイヤーを通じて構成された、系全体の分布状態とその履歴を保持する。

局所現象が系全体の分布にどの程度影響したかは、質量、広がり、依存構造、移動、曲率などを通じて `G_t / K_t` に反映される。

### H-DEPTが担当するもの

`G_t / K_t` に表れた系全体の性格を読む。

局所現象を完全に発見したり、その名称を当てたりすることは要求しない。局所現象が系全体を大きく再構成した場合には、結果としてH11の複数軸へ影響し得るが、それは局所ラベルを検出したからではなく、系全体の性格が変化したからである。

---

## 4. 局所固定化と系全体の固定化

「固定化」という言葉は、局所と全体で意味が異なる。

### 局所固定化

系の一部分だけが狭い状態へ収束している状況である。

その局所固定化が系全体の質量や構造にほとんど影響しないなら、H-DEPTが強く反応しないことは誤りではない。局所固定化の発見と詳細評価は下位レイヤーの役割である。

### 系全体としての固定化傾向

分布全体が狭まり、複数方向への移行余地が低下し、軸参加や構造多様性が全体として変化している状況である。

この場合は、H-DEPTが系全体の性格変化として読む対象になる。

したがって、新しいTask 4で検証すべきことは、次ではない。

> 小さな局所固定化をすべて見つけられるか。

検証すべきことは、次である。

> 同じ種類の局所変化でも、系全体を占める割合や全体構造への影響が増えるにつれて、上位観測の応答が適切に変わるか。

---

## 5. H11軸の意味

H11の各軸は、状態名、正解ラベル、目的関数、安全判定ではない。

それぞれは、系全体の分布と履歴に現れた性格を読むための記述座標・代理指標である。

### Stability

系全体の分布運動や曲率が低いことを表す。安全性そのものではない。

高Stabilityでも、閉じた状態、硬直、予測可能な崩壊などはあり得る。

### Exploration

系全体の分布の広がり、軸参加、モード構造、観測された移動などの代理指標である。

局所探索イベントを検出する軸ではない。値が低いことだけで局所固定化を断定しない。

### Coherence

系全体の内部的な幾何整合性を表す代理指標である。

高Coherenceは安全性を意味しない。境界へ集中した分布や、狭くまとまった分布が内部的には整合していることもある。

### StructuralDiversity

系全体の分布多様性と、閾値付きのモード分離を表す代理指標である。

すべての局所構造を発見したことの証明ではない。

### 共通原則

```text
高い値 ＝ 良い
低い値 ＝ 悪い
```

とは扱わない。

また、単一軸だけで安全・危険、健全・不健全、探索・固定化を判断しない。後段の統治判断では、複数軸の組合せ、信頼度、証拠被覆率、注意点、補助基底を合わせて読む必要がある。

## 6. 旧Task 4の問題

旧Task 4では、人工シナリオに「固定化」「構造化探索」「境界発散」などの名称を付け、その名称に対応するH11軸が一定方向へ一定幅以上動くことを科学的合格条件とした。

代表例は次である。

- 構造化探索ならExplorationとStructuralDiversityが固定化より0.08以上高い
- 境界発散ならCoherenceが広い安定分布より0.05以上低い
- 8仮説をすべて通過しなければ科学的不合格

この設計には次の問題がある。

1. シナリオ名を正解ラベルとして扱った
2. H11を局所・状態分類器のように扱った
3. Coherenceを安全性に近い意味で扱った
4. 理論から直接導かれていない数値差を必須条件にした
5. 単一軸方向を全体妥当性の合否へ使った

したがって、旧Task 4の8仮説による科学的合否は、H-DEPTの妥当性判断として使用しない。

旧Task 4で再利用できるのは、次の工学基盤である。

- 校正・検証・最終確認の分離
- seedと軌道識別子の分離
- 決定論的な軌道生成
- 校正用データだけを使う境界
- Task 2生成器との接続
- Task 3独立検証器との接続
- 成果物の決定論的生成
- CIとバイト一致確認

## 7. 新しいTask 4で許される検証

新しいTask 4では、次を中心に検証する。

### 7.1 系全体情報の保持

`G_t / K_t` の主要な分布形状、軸依存、広がり、移動、曲率、時間変化が、H11と生の特徴記録へ監査可能に反映されるか。

### 7.2 局所変化への過剰反応防止

系全体に対する占有率が小さい局所変化を、直ちに系全体の状態転換として扱わないか。

### 7.3 局所から全体への規模応答

同じ形の変化でも、影響する質量割合や全体構造への影響を段階的に増やしたとき、論理的に影響を受ける特徴・軸の応答が適切に変わるか。

ここで、すべての軸が単調に動くことを要求しない。どの特徴・軸にどの方向を期待できるかは、実行前に個別に根拠を示す。

### 7.4 連続性と閾値境界

入力分布をわずかに変えたとき、H11が不必要に飛躍しないか。閾値付き代理特徴で不連続が起きる場合は、その境界が記録・監査できるか。

### 7.5 証拠不足の明示

予約特徴、低被覆、短い履歴、代理特徴の限界が、利用不能、信頼度、証拠被覆率、注意点として正しく表れるか。

### 7.6 校正の健全性

校正値が検証・最終確認へ漏れていないか。値が過度に上限・下限へ張り付いて情報を失っていないか。校正データの選び方に対して過度に不安定でないか。

### 7.7 複数軸プロフィール

人間が付けた状態名を当てるのではなく、異なる系全体分布が、どのようなH11プロフィールとして表現されるかを比較する。

### 7.8 補助基底との関係

利用可能な場合は、H12_C、H20、PCA8、Random8などと比較し、H11が何を保持し、何を失うかを診断する。

これらの補助基底が固定5軸版で未実装の場合、その未実装自体をTask 1〜3の不具合とは扱わない。後続の比較設計として明示する。

---

## 8. 新しいTask 4で禁止する検証

次を科学的合格条件にしてはならない。

- すべての局所固定化を検出できること
- すべての局所探索を検出できること
- シナリオ名をH11から分類できること
- 高Coherenceを安全と読むこと
- 低Explorationを固定化の証明とすること
- 単一軸の方向だけで全体を合格・不合格にすること
- 理論的・統計的根拠のない差分閾値を必須条件にすること
- 小さな局所異常に大きな上位応答を要求すること
- Task 3の工学通過を科学的妥当性の証明とすること

## 9. Task 1〜3の扱い

Task 4-0では、Task 1〜3を変更しない。

### Task 1

正式入力、47特徴語彙、H11対応表、利用可能性、信頼度、注意点の契約を保持する。

次のTask 4-1で、これらが今回固定した「系全体の性格観測」という役割に沿っているかを意味監査する。

### Task 2

Task 1契約を実装する生成器として保持する。

### Task 3

Task 2がTask 1契約どおり計算したかを独立に検証する工学検証器として保持する。

Task 3の通過は、新しいTask 4でも前提条件になるが、科学的合格そのものではない。

---

## 10. Task 5の扱い

Task 4-0だけではTask 5へ進まない。

次に必要なのは、Task 4-1として、47特徴とH11対応表の意味監査を行うことである。

Task 4-1で問題がなければTask 1〜3を維持し、新しいTask 4の検証設計へ進む。

Task 4-1で対応表の意味的な問題が見つかった場合だけ、Task 1を版更新し、Task 2・3を追従修正する。
