# 固定5軸H11 — 再利用・置換表と射影分離

## 目的

この文書は、新しいH11を先に厳密固定するための仕様書ではない。
次の初期実装へ進むために、既存コードを「残す部分」と「置き換える部分」へ分け、旧H11方式を交換可能な位置へ隔離した記録である。

## 今回の完了範囲

- 既存コードの再利用・置換区分をファイル／責任単位で整理した。
- 旧H11射影を `h11_legacy_rc1.py` へ移した。
- `h11.py` を安定した差し替え境界にした。
- 既存builderは同じ `construct_h11(...)` を呼び続けるため、現在の出力値・成果物形式・Task 2〜4の挙動は変更しない。

## 今回は行わないこと

- 新しい11軸計算式の実装
- 47特徴の正式な再割当て
- 現在の科学結果の再計算
- PR #184の修復・マージ
- Task 5接続
- 旧方式の科学的正当化

## 再利用・置換表

| 対象 | 区分 | 次工程での扱い | 理由 |
|---|---|---|---|
| `canonical.py` の正本G/K読込み、因果的接頭部選択、履歴鎖 | 再利用 | 維持 | H11の意味射影と独立した入力・因果境界である |
| `geometry.py` の格子、分布状態、エントロピー、分布距離など | 条件付き再利用 | 計算単位ごとに小規模試験を維持して使用 | 意味付け前の幾何計算が中心だが、固定格子代理量は限定量として扱う必要がある |
| `features.py` の利用可能／利用不能記録形式 | 再利用 | 維持 | 欠測をゼロ埋めせず理由を保持する工学形式は有効 |
| `features.py` の共分散、固有値、有効次元、軸参加、分布距離、速度、加速度、曲率、回転、振動の計算 | 条件付き再利用 | H11割当てを外して初期特徴候補として使用 | 幾何・時間量として利用できるが、現在の軸意味との対応は未確定 |
| `features.py` の `reserved_prediction_subcontract` / `reserved_recovery_subcontract` による一律利用不能化 | 置換 | 新H11実装では使用しない | Predictability / Recoverability / Robustnessを専用観測器として扱った誤前提を含む |
| `configs/fixed5axis_hdept_feature_registry_rc1.json` の特徴ID、計算条件、履歴必要数 | 分割再利用 | 計算メタデータは候補として残し、H11意味・採点状態は再監査 | 計算定義と旧H11契約が同じ台帳に混在している |
| `configs/fixed5axis_hdept_h11_evidence_map_rc1.json` | 置換 | 旧方式の再現専用として保持 | 正特徴平均－負特徴平均、必須構成要素、3軸利用不能の根拠になっている |
| 旧 `h11.py` の校正特徴生成、構成要素平均、シグモイド、被覆率による利用判定 | 旧方式隔離 | `h11_legacy_rc1.py` に保持し、新実装では直接利用しない | 現在結果の再現には必要だが、次のH11意味射影の正本にはできない |
| 新 `h11.py` | 再利用 | H11射影の唯一の呼出し境界として維持 | feature builderや成果物出力を変更せず射影だけ交換できる |
| `builder.py` の G/K読込み → 特徴生成 → H11呼出し → 原子的成果物保存 | 再利用 | 維持 | H11の中身を差し替えても配管・成果物境界を保てる |
| `builder.py` の現在の科学claim文字列・旧evidence map hash | 後続修正 | 新H11を選択する段階で版・出所を明示 | 今回は後方互換のため変更しない |
| Task 3の正本非変更、ハッシュ、成果物集合、独立読込み、改ざん拒否 | 再利用 | 維持 | 射影数式と独立した完全性検証である |
| Task 3の旧H11数値再計算部分 | 置換 | 新射影と独立な別実装へ更新 | 現在は旧方式を独立再実装しているため、新方式の検証には使えない |
| Task 4の合成G/K生成、seed分割、連続性・飽和・軸間相関の検査 | 条件付き再利用 | 小規模対照試験後に必要部分だけ使用 | 検証基盤は使えるが、旧H11意味と合格条件から分離する必要がある |
| Random8 / PCA8 | 診断再利用 | 情報損失と圧縮傾向の比較だけに使用 | H11がRandom8を上回ることを科学合格条件にはしない |
| Task 4-3の8軸科学判定 | 使用不可 | 旧方式の参考結果としてのみ保存 | H11前提と3軸利用不能規則が誤っていた |

## 切り離した実行境界

現在の呼出しは次の形になった。

```text
builder.py
  └─ h11.construct_h11(...)
       └─ projection_idで実装を明示選択
            └─ h11_legacy_rc1.construct_h11_legacy_rc1(...)
```

既定の `projection_id` は旧RC1方式である。未知の方式へ暗黙にフォールバックせず、明示的に拒否する。

次の初期実装では、例えば次のように別モジュールを追加できる。

```text
h11_initial_v0.py
  └─ construct_h11_initial_v0(...)
```

その後 `h11.py` の選択肢へ追加する。`features.py`、`builder.py`、正本G/K読込み、成果物保存を同時に作り直す必要はない。

## 現在の到達点

- 工学基盤と旧H11意味射影の境界を分離した。
- 旧結果の再現経路を保持した。
- 新しいH11を実装する位置を一か所に限定した。
- 新H11の意味・特徴・重み・検証条件はまだ固定していない。
