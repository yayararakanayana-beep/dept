# 汎用関係場 G3 科学量互換・固定値非依存検証

## 1. 到達点

G3は、G2で作った固定5軸接続層の上で、既存RF-3〜RF-10の科学量を安定した識別子へ結び直し、値を失わず汎用成果物へ移せることを検証する。

G3が成立させる経路は次である。

```text
既存固定5軸 G_t / K_t
        ↓
既存 RF-3〜RF-10成果物            既存の科学計算・正本
        ↓ 各既存独立validator
G2構造台帳による識別子付与
        ↓
識別子順へ並べ替えたG3数値成果物
        ↓
元成果物からの独立再生成・全値比較
```

配列位置は保存上の位置に限定し、意味は次の識別子で復元する。

- `axis_id`
- `cell_id`
- `edge_id`
- `face_id`

G3成果物は、元配列のハッシュだけを保存するのではない。宣言された全数値配列を、識別子の正準順へ実際に並べ替えて保存する。独立validatorは元RF成果物から同じ処理をやり直し、配列のキー、形、型、全値を比較する。
成果物に同梱されたG3契約・検証プロファイルも、validatorへ指定したリポジトリ正本と完全一致しなければ停止する。

## 2. G3の範囲

G3で実装したもの。

- RF-3〜RF-10の既存独立validator呼出し
- 構造次元を明示した数値成果物対応表
- 軸・セル・辺・面識別子による正準並べ替え
- RF-5複合記述子の軸別・軸対別・全体量への展開
- 意味成果物の完全保存
- RF-3〜RF-10科学的不変条件の再検証
- 3軸・7軸など8種類の合成構造試験
- 能力不足・未来漏洩・不正質量・不正な予測統合・確率主張・作用選択の否定guard
- 決定論的・原子的成果物とSHA-256 manifest

G3で実装していないもの。

- RF-3〜RF-10の科学計算核の再実装
- 汎用生成 `G_t`・`K_t`
- 時間途中の構造変更または構造間対応
- ミクロ予測器
- マクロの実効ゲーム構造予測器
- 分布幾何シミュレーター
- 三予測器の実出力統合
- 予測精度評価
- 未知構造での精度主張
- 不可逆性の普遍的判定
- 作用評価・作用選択

したがってG3の科学的主張は、`固定5軸RF科学量の識別子付き互換`に限定する。

## 3. RF段階別の互換対象

| 段階 | 数値互換 | 意味互換 | 独立検証 |
|---|---|---|---|
| RF-3 | 辺流れ、観測差、再構成差、残差 | 再構成・不確実性 | 質量再構成と残差分解 |
| RF-4 | 代替候補流れ族 | 最小作用証明、代替解、solver・罰則・局所性監査 | 既存RF-4 validator＋候補配列全値 |
| RF-5 | 候補流れ、候補幅、代表流れ、共通流れ、軸流れ | 候補台帳、時間経路、診断、不確実性 | 全候補の質量再構成と区間順序 |
| RF-6 | 勾配、循環、調和、数値残差、節点ポテンシャル、面循環 | 候補指標、位相、診断、不確実性 | 再構成、循環発散、勾配面循環 |
| RF-7 | 形状、形状変化、流路幅、境界、場との整合 | 成分、形状候補、流路候補 | 最小・平均・最大の順序 |
| RF-8 | 軸流れ族、位置結合、時間差結合、新規駆動、残差 | 同一軸候補、新規駆動候補 | 候補幅、区間順序、残差との分離 |
| RF-9 | 並列リスク構造候補成分 | 候補、証拠、反証 | 数値・意味一致、単一点数禁止 |
| RF-10 | 未来結果は正解側としてのみ保存 | 凍結予測、標本台帳、評価、漏洩監査 | 最終RF-9行限定、未来読込前凍結 |

RF-10の未来結果は関係場観測ではない。`future_target_only`として明示し、予測特徴へ戻さない。

## 4. 合成構造試験

G3は次の構造を、同じG2構造validatorへ通す。

1. 3軸・軸ごとに異なる区分数
2. 7軸・二値直積
3. 不規則接続
4. 複数連結成分
5. 面なし循環
6. 面を持つが穴も残る構造
7. 座標なし構造
8. セル・辺・面の保存順を変えた同一構造

検証する内容。

- 向き付き接続行列
- `B1 × B2 = 0`
- 連結成分数
- 一次Betti数
- 構造能力
- 識別子順変更前後の正準構造署名

この試験は、未知構造の予測精度や汎用G/K生成を証明しない。

## 5. 有効性と否定guard

### 面なし構造

面がない場合、調和成分を有効値として出せない。`unavailable`または`withheld`とし、値を空にする。

### 座標なし構造

座標がない場合、軸方向を捏造しない。接続構造の量だけを継続できる。

### 不正質量

負質量または質量和の不一致を、切詰めや再正規化で修復しない。入力違反として停止する。

### 予測器不一致

将来の三予測器が`conflicting`となった場合、平均予測や統合単一点数を作らない。不一致状態を保持する。

### 確率

独立校正、支持数、校正成果物参照が揃わない値を確率と呼ばない。

### シミュレーター

将来の分布幾何シミュレーター成果物に、作用採用、最適作用、望ましさ、総効用を含めない。

G3成果物内で実行して記録するguardは、面なし調和、座標なし方向、未来キー、不正質量、
予測器不一致平均、未校正確率、シミュレーター作用選択の7件である。構造ハッシュ不一致、
K_t内構造変更、未知識別子の3件はG2の責務なので、G3成果物は「G2継承必須」と記録し、
結合CIでG2とG3の否定試験をともに実行する。G3成果物単体で後者3件を実行済みとは記録しない。

## 6. 実行例

合成構造試験だけを実行する。

```bash
python scripts/generic_relation_field_g3.py synthetic
```

G3正式成果物の生成には、G2構造成果物、固定5軸格子、正本軌道、RF-3〜RF-10成果物、RF-10 case manifestをすべて明示する。

```bash
python scripts/generic_relation_field_g3.py build \
  --structure <g2_structure> \
  --grid <rf2_grid> \
  --trajectory <canonical_trajectory> \
  --rf3 <rf3_artifact> \
  --rf4 <rf4_artifact> \
  --rf5 <rf5_artifact> \
  --rf6 <rf6_artifact> \
  --rf7 <rf7_artifact> \
  --rf8 <rf8_artifact> \
  --rf9 <rf9_artifact> \
  --rf10 <rf10_artifact> \
  --rf10-case-manifest <case_manifest> \
  --output <g3_output>
```

独立再検証は`validate`で同じ正本を指定する。

## 7. 停止条件

次の場合は推測補正せず停止する。

- G2構造成果物のmanifestまたは構造ハッシュ不一致
- 同梱G3契約・検証プロファイルと指定正本の不一致
- G2構造とRF-2格子manifestの不一致
- RF-3〜RF-10のどれかの既存validator失敗
- 必須数値配列の欠落
- 対応表にない数値配列の出現
- 構造次元と台帳件数の不一致
- 意味成果物の欠落
- 科学的不変条件の許容差超過
- RF-10の因果境界違反
- 合成構造の不変条件違反
- 元成果物の変更
- 出力先の既存

## 8. 次段階との関係

## 12. 順序付き実装手順

1. G3契約と検証プロファイルを検証し、成果物検証時は指定正本との完全一致も確認する。
2. G2構造成果物をmanifest付きで検証する。
3. G2構造とRF-2格子のmanifestを照合する。
4. 元成果物全体のハッシュを凍結する。
5. RF-3〜RF-10の既存独立validatorをすべて実行する。
6. 軸・セル・辺・面の識別子を正準順へ並べる。
7. 対応表で宣言した全数値配列を実際に並べ替える。
8. RF-5複合記述子を軸別・軸対別・全体量へ展開する。
9. 宣言した意味成果物を完全保存する。
10. 汎用成果物から科学的不変条件を計算する。
11. 合成構造8件をG2構造validatorへ通す。
12. 否定guardを実行する。
13. 元成果物ハッシュが変わっていないことを確認する。
14. 原子的にG3成果物を保存する。
15. 独立validatorが1〜13を再実行して全値を照合する。

## 13. 重要要件契約

### 13.1 科学値の完全な物質化

- 要件: 互換判定は件数またはハッシュだけでなく、数値配列の実値を保持する。
- 必須実装: 対応表の全NPZキーを列挙し、各構造次元を宣言し、正準ID順へ並べ替えた配列を`numeric_payloads.npz`へ保存する。
- 禁止する近道: 元ファイルSHA、配列件数、`compatible=true`だけを保存する。
- 検証: 独立validatorが元RF配列から再生成し、キー・形・型・全値を比較する。manifestを作り直した数値改ざんも否定試験で拒否する。

### 13.2 配列位置と意味識別の分離

- 要件: 固定5軸の配列位置を意味識別として使わない。
- 必須実装: G2台帳IDを保存し、Unicode正準ID順へ実際に並べ替える。
- 禁止する近道: 元配列をそのままコピーし、ID一覧だけ後付けする。
- 検証: 元辺順と正準辺順が異なることを確認し、ID joinで再構成した値とG3値を比較する。合成構造の順序変更前後で構造署名を比較する。

### 13.3 科学的不変条件

- 要件: 値が保存された事実だけでなく、科学量間の関係が維持される。
- 必須実装: RF-3質量再構成、RF-5全候補再構成、RF-6分解、RF-7/8候補区間、RF-9数値・意味一致、RF-10因果境界を汎用成果物から再計算する。
- 禁止する近道: 既存`validation.json`の`passed`を転記する。
- 検証: G3独立validatorが汎用数値配列と接続演算子から再計算し、許容差を各既存契約から読む。

### 13.4 固定値非依存

- 要件: G2構造中核が固定5軸の件数へ依存しない。
- 必須実装: 3軸、7軸、異区分、不規則、複数成分、面なし、穴、座標なし、順序変更を含む8件を動的生成する。
- 禁止する近道: 固定5軸の一部だけを切り出して別構造と呼ぶ。
- 検証: 各構造の実件数、連結成分、一次Betti数、`B1 × B2`を実計算する。

### 13.5 RF-10因果境界

- 要件: 未来結果を予測特徴へ混ぜない。
- 必須実装: 既存RF-10 validatorを再実行し、最終RF-9行限定、予測凍結後の未来読込、正解側限定をG3でも確認する。
- 禁止する近道: 漏洩監査の自己申告だけを保存する。
- 検証: RF-10 case manifestから予測・未来結果・標本台帳を再生成し、G3意味成果物と完全比較する。

### 13.6 能力不足と禁止出力

- 要件: 面・座標・支持数・予測器整合性が不足する場合に値を捏造しない。
- 必須実装: 面なし調和、座標なし方向、未来キー、不正質量、不一致平均、未校正確率、シミュレーター作用選択を拒否するguardを持つ。
- 禁止する近道: 0、空文字、既定確率、平均値で処理を継続する。
- 検証: 各guardに実際の拒否否定試験を置く。

## 18. 近道監査

### 科学量互換

1. 最安の偽実装: `compatible=true`と元manifest SHAだけを保存する。
2. 固定値・空値・`true`・`0`で代替できるか: 全数値配列を独立比較するため不可。
3. producerが自己認証できるか: validatorが元RF成果物から再生成するため不可。
4. smokeが正式分岐を飛ばせるか: RF-3〜RF-10正式builderとvalidatorをすべて実行するため不可。
5. 否定試験: 数値改ざん後にmanifestを再生成しても拒否する。
6. 設定が唯一の正本か: ファイル対応と許容差参照は検証プロファイルのみ。
7. 未実装を既定値で隠せるか: 未宣言配列または欠落配列で停止する。
8. 最小の偽実装が試験を通るか: 通らない。

### 固定値非依存

1. 最安の偽実装: 固定5軸配列を小さく切って3軸・7軸と表示する。
2. 固定値・空値・`true`・`0`で代替できるか: 構造を動的生成し実件数と行列階数を計算するため不可。
3. producerが自己認証できるか: G2構造validatorとBetti数計算を別に実行するため不可。
4. smokeが正式分岐を飛ばせるか: 8件すべて必須。
5. 否定試験: 能力不足の偽値を拒否する。
6. 設定が唯一の正本か: 合成ケースは検証プロファイルのみ。
7. 未実装を既定値で隠せるか: 必須case ID集合が欠けると契約違反。
8. 最小の偽実装が試験を通るか: 通らない。

### RF-10因果境界

1. 最安の偽実装: 漏洩なしの`true`だけを転記する。
2. 固定値・空値・`true`・`0`で代替できるか: case manifestから予測snapshotと未来結果を再生成するため不可。
3. producerが自己認証できるか: 既存RF-10 validatorとG3 validatorが再計算するため不可。
4. smokeが正式分岐を飛ばせるか: 同一接頭部・異未来2ケースを実行する。
5. 否定試験: 未来特徴キーを拒否する。
6. 設定が唯一の正本か: RF-10契約を参照する。
7. 未実装を既定値で隠せるか: snapshot、標本台帳、評価成果物の欠落で停止する。
8. 最小の偽実装が試験を通るか: 通らない。

すべての重要出力について、最小の偽実装は試験を通らない。
